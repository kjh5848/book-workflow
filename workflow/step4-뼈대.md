# STEP 4. 뼈대 세우기 — "목차와 코드 실습 배치"

> **목적**: 코드 실습 분류 + 독자 여정 + 목차 + 갭 분석 + 기술 매핑을 한 번에.
> **스킬**: B3.코드-태거, B4.계층-생성기, B5.난이도-곡선기, B6.갭-분석기, C4.제목-생성기, D6.분량-계산기
> **검토**: [인사이트 검토] + [감수 검토]
> **산출물**: `planning/outline.md`

---

## 흐름

```
--- 코드 실습 분류 ---
1. B3.코드-태거 → 버전별 파일에 [실습/설명/참고] 태그
2. B4.계층-생성기 → 태그 포함 파일 트리
3. "이 버전 핵심 주제가 뭐야?" → 실습 코드 결정
4. 저자 조정

--- 여정 설계 ---
5. "독자가 먼저 알아야 할 건?" → 출발점
6. "아하 순간이 언제?" → 전환점 위치
7. "제일 헷갈릴 구간은?" → 난이도 스파이크
8. B5.난이도-곡선기 → 난이도 시각화

--- 목차 초안 ---
9. C4.제목-생성기 → 챕터 제목 후보
10. D6.분량-계산기 → 페이지 배분
11. 목차 초안 구성 + 기술 매핑

--- 갭 분석 ---
12. B6.갭-분석기 → 도메인 표준 대비 누락 주제 [필수/권장/선택]
13. 저자에게 반영 여부 확인

--- 최종 조정 ---
14. "빠진 거 있어? 빼도 되는 거 있어?"
15. "완성 코드와 다르게 가르치고 싶은 부분 있어?"
16. 저자 조정
    ↓
[인사이트 검토] → 분량, 난이도 배분, 제목 질문
    ↓
[감수 검토]
  - 기술 감수자: 빠진 기술은? 순서 최적화 제안
  - 독자 대변인: 챕터 수가 적절? 완주 가능?
  - 이야기 편집장: 챕터 제목이 호기심을 유발하나?
    ↓
저자 확인 → planning/outline.md 생성
```

|------|------|------|----------|

| 설명 | [설명] | 중요하지만 핵심 아닌 코드 | 코드 읽고 이해 |
