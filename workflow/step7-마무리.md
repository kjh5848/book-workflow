### 서문 톤 선택지

서문 작성 전, 저자에게 **톤 선택지**를 제안한다.

```
서문 톤을 골라주세요:

  A) 독자 공감형 — "이런 고민, 해보신 적 있으시죠?" 식으로 독자 상황에서 시작.
     저자 개인 이야기 없이 독자가 왜 이 책이 필요한지에 초점.
  B) 저자 경험형 — "저도 이걸 처음 해봤습니다" 식으로 저자 경험에서 시작.
     seed.md의 저자 경험이 구체적일 때 적합.
  C) 문제 제기형 — "기존 RAG 자료에는 이런 게 없습니다" 식으로 시장 빈자리에서 시작.
     차별점을 강조하고 싶을 때.
  D) 대화형 — 독자에게 편하게 말 거는 느낌. "솔직히 말씀드릴게요."
     친근하고 가벼운 톤.
  ✏️ 직접 쓸게
```

### 서문 작성 규칙

- **저자 중립**: seed.md의 동기/경험이 "AI 범용 작성"이면, 서문에서 저자 개인 경험을 전제하지 않는다
- **톤 일관성**: 선택한 톤을 서문 전체에 일관되게 유지한다
- **구조**: 도입(왜 이 책?) → 차별점(이 책만의 것) → 읽는 방법 → 클로징

```
1. 질문 1~3 수집
2. 서문 작성
3. 에필로그 작성

5. C4.제목-생성기 → 최종 제목 + 부제목 확정
6. D6.분량-계산기 → 전체 분량 확인
    ↓
[감수 검토]
  - 기술 감수자: 부록에 넣을 심화 자료 제안
  - 독자 대변인: 에필로그가 독자에게 용기를 주나?
  - 이야기 편집장: 서문이 진정성 있나?
    ↓
저자 확인 → 최종 산출물 생성
```
