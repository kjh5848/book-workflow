# STEP 6. 프롤로그 + 로드맵 — "숲을 보여준다"

> **목적**: 모든 챕터가 끝난 후, 코드 없이 전체를 한 눈에.
> **스킬**: C2.요약기
> **검토**: [감수 검토]
> **산출물**: `book/front/prologue.md`, `book/front/roadmap.md`

---

## 흐름

```
1. C2.요약기 → 각 챕터에서 핵심 개념/비유만 추출
2. 시나리오 흐름대로 하나의 이야기로 연결 → 프롤로그
3. 버전별 성장 과정을 시각적으로 → 로드맵
    ↓
[감수 검토]
  - 기술 감수자: 핵심 개념이 빠지진 않았나?
  - 독자 대변인: 프롤로그만 읽고 책을 사고 싶어지나?
  - 이야기 편집장: 하나의 완결된 이야기인가?
    ↓
저자 확인 → book/front/prologue.md, roadmap.md 생성
```

---

## 프롤로그 작성 규칙

- **코드 없음**: 프롤로그에는 코드가 등장하지 않는다
- **비유만 사용**: 각 챕터의 핵심 비유를 하나의 이야기로 엮는다
- **전체 여정 미리보기**: 독자가 앞으로 어떤 여정을 떠나는지 보여준다
- **호기심 유발**: 읽고 나면 본문이 궁금해져야 한다

## 로드맵 작성 규칙

- **시각적**: 버전별 성장을 다이어그램이나 리스트로 보여준다
- **성과 중심**: 각 버전에서 "뭐가 보이는지"를 강조한다
- **한 페이지**: 로드맵은 한 페이지 이내로 간결하게
