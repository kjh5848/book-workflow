# STEP 2. 코드 해부 — "재료가 뭐가 있지"

> **목적**: 완성 코드를 의도 필터로 분석. 핵심 기능 추출.
> **전제**: 코드 워크플로우 산출물이 code/ 폴더에 있거나 GitHub URL 제공.
> **스킬**: A1.구조-스캐너, A2.기능-추출기, A3.기술스택-탐지기, A4.의존성-매퍼
> **검토**: [인사이트 검토] + [감수 검토]
> **산출물**: `planning/code-analysis.md`

---

## 흐름

```
--- AI 자동 분석 ---
1. A1.구조-스캐너 → 프로젝트 트리
2. A2.기능-추출기 → 기능 목록
3. A3.기술스택-탐지기 → 기술 스택
4. A4.의존성-매퍼 → 컴포넌트 의존성

--- 의도 필터 적용 ---
5. planning/seed.md 의도 필터 적용 → 의도 안/밖 분류
6. "이 기능 목록 맞아?" → 핵심 기능 확인
7. "이 기능 중에 안 보여줘도 되는 건?" → 범위 확정
    ↓
[인사이트 검토] → 코드에서 발견한 이슈 질문
    ↓
[감수 검토]
  - 기술 감수자: 빠진 핵심 개념은? 기술 의존성은?
  - 독자 대변인: 첫 버전이 너무 어렵지 않나?
    ↓
저자 확인 → planning/code-analysis.md 생성
```

---

## 인사이트 질문 예시

| 유형 | 예시 |
|------|------|
| 기술 발견 | "코드에 `@Transactional`이 있는데, 트랜잭션을 설명할 거야? 한다면 JPA보다 앞에 와야 할 수도 있어" |
| 범위 확인 | "이 코드들은 의도 밖이라 분석에서 뺄게. 혹시 다시 넣고 싶은 거 있어?" |
| 선행 개념 | "이 기능을 설명하려면 독자가 OO을 알아야 하는데, 배경지식에 포함되는 거야?" |
