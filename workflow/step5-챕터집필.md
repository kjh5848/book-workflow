## 한 챕터의 흐름

```
--- 변수 수집 ---
1~8번 질문 → [인사이트 검토]: 놓친 부분 질문

--- 이야기 파트 작성 ---
9. C1.비유-생성기 → 핵심 개념 비유
10. 이야기 작성 (도입 → 비유 → 시나리오 → 시행착오)
11. C3.브릿지-생성기 → 다음 챕터 연결
12. C2.요약기 → "이것만은 기억하자"

--- 기술 파트 작성 ---
13. C5.용어-정의기 → 비유→정식 정의 테이블
14. outline.md에서 코드 분류 가져와서 기술 파트 작성

--- 자체 검증 ---
15. D1.용어-탐지기 → 이야기 파트 어려운 용어
16. D2.톤-검사기 → 대화체 유지 확인
17. D3.파트-분리-검증기 → 이야기/기술 분리 확인
18. D4.포맷-검증기 → 상수 준수 확인

--- 의도 검증 ---
19. [의도감시 검토] → 의도 이탈/코드 성격/깊이 체크
    → 위반 시 수정

--- 감수 ---
20. [감수 검토] → 3인 편집장 리뷰
  - 기술 감수자: 기술 설명이 정확? 더 좋은 예시?
  - 독자 대변인: 이 챕터만 읽어도 이해되나?
  - 이야기 편집장: 도입 이야기가 몰입감 있나?
    ↓
저자 확인 → 수정 → chapters/NN-제목.md 저장
    ↓ 다음 챕터로 반복
```

## 챕터 내부 구조 (상수)

```
┌─────────────────────────────────────┐
│  이야기 파트 (코드 없음)              │
│  - 도입 (경험, 상황)                 │
│  - 개념 (비유 1~2문장)               │
│  - 시나리오 진행 (이번 버전에서 뭘 바꾸나)│
│  - "이것만은 기억하자" + 다음 예고     │
├─────────────────────────────────────┤
│  기술 파트 (이야기 끝난 후)            │
│  - 용어 정리 (비유→정식 정의)          │
│  - 파일 계층 구조 [실습/설명/참고]      │
│  - 실습 코드 + 설명 코드              │
│  - 더 알아보기                       │
└─────────────────────────────────────┘
```
