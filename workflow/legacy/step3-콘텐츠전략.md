---

**Q2. 코드는 어떻게 보여줄까요?**

> 아래 중에서 골라주세요:
> - (A) 복사-붙여넣기 가능 — 그대로 복사해서 실행하면 바로 동작하는 완전한 코드
> - (B) 핵심만 스니펫 — 핵심 로직만 보여주고, 전체 코드는 GitHub 참조
>
> (어떤 방식이든 코드에 대한 스토리텔링 설명은 항상 포함됩니다)

---

**Q3. 각 챕터에 넣고 싶은 부가 요소가 있나요?**

> 원하는 것을 모두 골라주세요 (복수 선택 가능):
> - (A) 챕터 끝 요약 — "이 챕터에서 배운 것" 정리
> - (B) 연습 문제 — 직접 해볼 수 있는 과제 1~3개
> - (C) 트러블슈팅/FAQ — 자주 발생하는 에러와 해결법
> - (D) 없음 — 본문만으로 충분
>
> 잘 모르겠으면 "알아서 정해줘"라고 해주세요.

---

**Q4. 스크린샷이나 이미지는 얼마나 넣을까요?**

> - (A) 많이 — 설치 과정, 실행 결과, UI 등 가능한 한 많이
> - (B) 적당히 — 핵심적인 부분만 선별적으로
> - (C) 최소한 — 텍스트와 코드 위주
>
> (이미지는 자리만 잡아두고 나중에 추가합니다)

---

## 진행 규칙

1. Q1~Q2를 먼저 묻고, 답변을 받은 뒤 Q3~Q4를 진행한다.
2. 모든 답변은 `answers.md`에 즉시 저장한다.
3. **AI 창의적 추가 질문**: 고정 질문이 끝난 후, 앞 단계 답변을 종합해 AI가 추가 질문을 생성한다. 예시:
   - "이 페이지 수에 맞추려면 OO 부분을 좀 압축해야 할 수도 있는데, 어떤 부분이 제일 중요할까요?"
   - "코드 실행 결과를 보여줄 때, 터미널 화면도 스크린샷으로 넣을까요?"
4. 완료 후 "3단계 완료! 마지막으로 책의 톤과 제목을 정해볼까요?" 안내.
