# 구성 생성 가이드

`구성 생성` 명령 시 이 가이드를 따라 책의 전체 목차를 생성한다.

## 입력

- `projects/[책이름]/answers.md`의 4단계 답변 전체

## 생성할 내용

### 1. 책 제목 후보 3개 + 부제목
- answers.md의 제목 아이디어와 셀링 포인트를 참고해서 제목 후보 3개를 제안한다.
- 각 제목에 어울리는 부제목도 함께 제안한다.
- 제목 기준:
  - 주제가 명확히 드러날 것
  - 독자 수준에 맞는 단어 사용
  - 30자 이내 권장

### 2. 전체 목차
챕터별로 다음을 포함한다:
- **챕터 번호 + 제목**
- **개요** (1~2줄): 이 챕터에서 다루는 내용 요약
- **핵심 키워드**: 챕터에서 다루는 주요 개념 2~3개

목차 구성 원칙:
- 첫 챕터는 "들어가며" (프로젝트 소개, 환경 설정)
- 마지막 챕터는 "마무리" (전체 정리, 다음 학습 안내)
- 중간 챕터는 학습 난이도 순으로 배치
- 각 챕터는 이전 챕터의 내용을 기반으로 자연스럽게 이어져야 함
- 100페이지 이하 기준으로 5~10개 챕터가 적절

### 3. 챕터별 예상 페이지 배분
- 전체 목표 페이지 수를 챕터별로 배분한다.
- "들어가며"와 "마무리"는 짧게 (3~5페이지)
- 핵심 챕터는 길게 (10~15페이지)
- 전체 합계가 목표 페이지 수를 넘지 않도록 한다.

### 4. 챕터별 포함할 코드 파일 매핑
- GitHub 레포나 code/ 폴더의 코드 파일을 각 챕터에 매핑한다.
- 코드가 아직 없으면 "필요한 코드 파일"을 목록으로 정리한다.
- 형식: `챕터 N → 파일1, 파일2`

### 5. 학습 흐름도
챕터 간의 선수/후속 관계를 명시한다:
```
챕터 1 (환경 설정) → 챕터 2 (기본 개념)
챕터 2 → 챕터 3 (핵심 기능 1)
챕터 2 → 챕터 4 (핵심 기능 2)
챕터 3, 4 → 챕터 5 (통합)
```

## 출력 형식

생성된 목차는 아래 형식으로 `outline.md`에 저장한다:

```markdown
# [책 제목]
> [부제목]

## 제목 후보
1. [후보 1] - [부제목]
2. [후보 2] - [부제목]
3. [후보 3] - [부제목]

**선택된 제목**: (사용자 확인 후 기입)

## 목차

### 00. 들어가며
- **개요**: [설명]
- **페이지**: [N]p
- **키워드**: [키워드들]

### 01. [챕터 제목]
- **개요**: [설명]
- **페이지**: [N]p
- **키워드**: [키워드들]
- **관련 코드**: [파일 목록]

...

## 페이지 배분 요약

| 챕터 | 제목 | 페이지 |
|------|------|--------|
| 00 | 들어가며 | Np |
| 01 | ... | Np |
| ... | ... | ... |
| **합계** | | **Np** |

## 학습 흐름도

[흐름도]
```

## 후속 작업

1. outline.md를 사용자에게 보여주고 검토를 요청한다.
2. 수정 사항이 있으면 반영한 뒤 다시 확인받는다.
3. 최종 확정되면 "제목이 확정되었습니다. '챕터 작성 [번호]'로 집필을 시작하세요!" 안내.
