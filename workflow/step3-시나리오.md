# STEP 3. 시나리오 + 버전 설계 — "어떤 순서로 이야기하지"

> **목적**: 책 전체를 관통하는 이야기 + 코드를 버전별로 쪼개기.
> **스킬**: A5.diff-생성기, B1.기능-정렬기, B2.스냅샷-설계기
> **검토**: [인사이트 검토] + [감수 검토]
> **산출물**: `planning/scenario.md` + `versions/`

---

## 질문 (변수 수집)

| # | 질문 | 인사이트 질문 | 예시 |
|---|------|-------------|------|
| 1 | "책 전체를 관통하는 프로젝트/상황이 있어?" | "'블로그를 만든다'가 아니라 왜 만드는지 동기가 있어야 해. 동기가 있어야 독자가 감정이입해" | "주니어가 사수 없이 블로그를 만들어가는 여정" |
| 2 | "제일 중요한 전환점은?" | "전환점은 '아하!' 순간이야. 쉬운 방법으로 했다가 한계를 느끼고 제대로 된 방법을 배우는 지점이 좋아" | "서버 재시작했더니 데이터가 다 날아간 순간" |
| 3 | "각 버전 끝에 독자 눈에 뭐가 보여?" | "눈에 보이는 게 있어야 계속 따라와. 내부 리팩터링만 하는 버전은 독자가 지쳐" | v0.1: "Hello World" / v0.2: 글 목록 페이지 |
| 4 | "순서 바꾸고 싶은 거 있어?" | "이 순서로 따라갈 때 중간에 포기할 지점이 있나? v0.3→v0.4 사이 난이도가 뛰면 쉬어가는 버전을 끼워야 해" | — |

## 흐름

```
--- 시나리오 ---
1. 질문 1: 시나리오 (동기 포함)
2. "처음부터 끝까지 이야기해줘" → 전체 흐름
3. 질문 2: 전환점
4. "마지막에 뭐가 완성돼?" → 최종 결과물

--- 버전 분해 (시나리오 기반) ---
5. B1.기능-정렬기 → 학습 순서 정렬
6. B2.스냅샷-설계기 → 버전별 스냅샷
7. A5.diff-생성기 → 버전 간 차이
8. 질문 3: 버전별 성과
9. 질문 4: 저자 조정

--- 버전별 예제 코드 생성 ---
10. 완성 코드에서 버전별 코드 추출/축소 → versions/ 폴더에 저장
    ↓
[인사이트 검토] → 전환점, 난이도 곡선, 포기 구간 질문
    ↓
[감수 검토]
  - 기술 감수자: 버전 순서가 기술 의존성에 맞나?
  - 독자 대변인: 시나리오에 감정이입 가능? 포기 구간은?
  - 이야기 편집장: 전환점이 극적인가? 긴장감 있나?
    ↓
저자 확인 → planning/scenario.md + versions/ 생성
```
