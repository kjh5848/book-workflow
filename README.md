# book-workflow

기술 서적(100페이지 권장)을 이야기처럼 쓰는 워크플로우 시스템.

저자(도메인 전문가)와 하나의 AI(Claude)가 대화하며 책을 완성한다.

## 시작하기

### 1. 완성 코드 준비

책을 쓰기 전에 **완성 코드가 먼저 준비**되어 있어야 한다.

- **로컬 경로**: `code/` 폴더에 직접 넣기
- **GitHub URL**: 저장소 URL을 알려주면 AI가 분석

### 2. 새 책 만들기

`새 책 만들기`를 입력하면 프로젝트가 생성된다.

```
projects/[책이름]/
├── code/               ← 여기에 완성 코드를 넣는다
├── planning/           ← STEP 1~4 산출물
├── chapters/           ← STEP 5 산출물
├── versions/           ← 버전별 예제 코드 (STEP 3에서 생성)
│   ├── v0.1/
│   ├── v0.2/
│   └── ...
├── book/               ← 최종 원고
├── questions/          ← 인사이트 질문 장바구니
├── review/             ← 검토 피드백
└── progress.json       ← 상태 관리
```

### 3. STEP 순서대로 진행

| 명령어 | STEP | 하는 일 |
|--------|------|---------|
| `새 책 만들기` | 프로젝트 생성 + STEP 1 | 6개 질문으로 책의 의도 확립 |
| `코드 분석` | STEP 2 | `code/` 안의 완성 코드를 의도 필터로 분석 |
| `시나리오 설계` | STEP 3 | 시나리오 + 버전 분해 + 버전별 예제 코드 |
| `뼈대 세우기` | STEP 4 | 코드 실습 분류 + 목차 + 갭 분석 |
| `챕터 작성 [번호]` | STEP 5 (반복) | 이야기 파트 + 기술 파트 집필 |
| `프롤로그 생성` | STEP 6 | 코드 없이 전체 개념을 이야기로 |
| `마무리` | STEP 7 | 서문, 맺음말, 부록, 최종 제목 |
| `현재 상태` | — | 진행률 확인 |
| `검토 [챕터]` | — | 작성된 챕터 검토 재실행 |

## 구조

```
.claude/skills/    ← 스킬 5개 카테고리 (writing, planning, code, visual, review)
workflow/          ← 7 STEP 실행 가이드
design/            ← 설계 문서 (v1~v3)
projects/          ← 프로젝트별 산출물
```

## 워크플로우 (7 STEP)

```
Phase 1 ── 의도 확립
  STEP 1. 씨앗              "이 책은 뭐다"

Phase 2 ── 재료 파악
  STEP 2. 코드 해부          "재료가 뭐가 있지"

Phase 3 ── 이야기 설계
  STEP 3. 시나리오 + 버전     "어떤 순서로 이야기하지"
  STEP 4. 뼈대 세우기         "목차와 코드 실습 배치"

Phase 4 ── 집필
  STEP 5. 챕터 집필 (반복)    "쓴다"

Phase 5 ── 완성
  STEP 6. 프롤로그 + 로드맵   "숲을 보여준다"
  STEP 7. 마무리              "서문, 맺음말, 부록"
```
