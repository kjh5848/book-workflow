# 에이전트-스킬 구조 v2 (집필 워크플로우)

> **이 문서는 `워크플로우-설계-v2.md`와 한 쌍이다.**
> 워크플로우-설계-v2.md가 "무엇을 어떤 순서로 하는가(STEP)"를 정의하고,
> 이 문서는 "누가 어떤 도구로 하는가(에이전트+스킬)"를 정의한다.

> **코드 워크플로우**는 완성 코드를 **만드는** 단계다 (별도 설계).
> **집필 워크플로우**(이 문서)는 완성 코드를 **분석하고 책을 쓰는** 단계다.
> 코드 문법, 코드 컨벤션, 코드 작성 자체에 대한 스킬은 이 문서에 포함하지 않는다.

---

## 설계 원칙

- **에이전트**: 대화를 이끌고, 판단하고, 다음 행동을 결정한다
- **스킬**: 하나의 작업만 수행하고 결과를 돌려준다. 판단하지 않는다
- **스킬은 원자(atomic)**: 하나의 스킬은 하나의 일만 한다. 여러 에이전트가 조합해서 사용할 수 있다

## 스킬 카탈로그 (22개)

### A. 코드 분석 스킬 (5개)

완성 코드를 읽고 정보를 추출하는 스킬.
코드를 **만드는** 게 아니라 **분석하는** 스킬이므로 집필 워크플로우에 포함한다.

#### A1. 구조-스캐너

> 프로젝트의 디렉토리/파일 구조를 트리로 출력한다.

| 항목 | 내용 |
|------|------|
| **입력** | 프로젝트 경로 또는 GitHub URL |
| **출력** | 디렉토리 트리 (파일명 + 한 줄 설명) |
| **하는 일** | 파일/폴더 구조를 읽고 트리로 정리 |
| **안 하는 일** | 코드 내용 분석, 기능 판단 |

```
입력: code/
    ↓
출력:
  code/
  ├── src/main/java/
  │   ├── controller/
  │   │   ├── PostController.java
  │   │   └── LoginController.java
  │   ├── service/
  │   │   ├── PostService.java
  │   │   └── AuthService.java
  │   ├── domain/
  │   │   ├── Post.java
  │   │   └── User.java
  │   └── repository/
  │       ├── PostRepository.java
  │       └── UserRepository.java
  └── src/main/resources/
      ├── templates/
      └── application.yml
```

#### A2. 기능-추출기

> 코드에서 사용자 관점의 기능 목록을 추출한다.

| 항목 | 내용 |
|------|------|
| **입력** | 코드 파일들 |
| **출력** | 기능 목록 (기능명 + 관련 파일 + 한 줄 설명) |
| **하는 일** | 코드를 읽고 "사용자가 할 수 있는 것" 단위로 기능 추출 |
| **안 하는 일** | 기술 스택 분석, 의존성 판단 |

```
입력: PostController.java, LoginController.java, ...
    ↓
출력:
  1. 글 목록 보기 — PostController.list() → post-list.html
  2. 글 작성 — PostController.create() → post-form.html
  3. 글 수정 — PostController.update()
  4. 글 삭제 — PostController.delete()
  5. 로그인 — LoginController.login() → login.html
  6. 로그아웃 — LoginController.logout()
  7. 회원가입 — LoginController.register() → register.html
```

#### A4. 의존성-매퍼

> 코드 컴포넌트 간 의존 관계를 매핑한다.

| 항목 | 내용 |
|------|------|
| **입력** | 코드 파일들 |
| **출력** | 의존성 그래프 (A → B: "A가 B를 사용") |
| **하는 일** | import, 주입, 호출 관계를 추적하여 의존 방향 정리 |
| **안 하는 일** | 기능 분류, 학습 순서 판단 |

```
입력: 전체 코드
    ↓
출력:
  PostController → PostService → PostRepository → Post(Entity)
  LoginController → AuthService → UserRepository → User(Entity)
  AuthService → HttpSession
  PostController → HttpSession (로그인 체크)

  의존성 메모:
  - PostController가 HttpSession을 쓰므로, 인증 챕터가 CRUD 챕터 뒤에 와야 함
  - AuthService가 UserRepository를 쓰므로, JPA 개념이 인증보다 먼저 나와야 함
```

#### B6. 갭-분석기

> 도메인 표준 커리큘럼과 현재 목차를 비교하여 누락된 주제를 찾아낸다.

| 항목 | 내용 |
|------|------|
| **입력** | 현재 `outline.md` + 도메인 키워드 + `seed.md` |
| **출력** | 누락 주제 목록 (필수/권장/선택 우선순위) |
| **하는 일** | 도메인 표준 커리큘럼 도출 → 현재 목차와 비교 → 빠진 항목 식별 → 우선순위 분류 |
| **안 하는 일** | 목차 수정, 챕터 내용 작성 |
| **사용 시점** | 워크플로우-설계-v2.md STEP 4 갭 분석 단계 |

우선순위 기준:
| 등급 | 조건 |
|------|------|
| [필수] | 이거 없으면 독자가 핵심 내용을 이해할 수 없음 |
| [권장] | 있으면 책의 차별점이 됨, 독자 만족도 상승 |
| [선택] | 심화/부록 처리 가능, 분량 고려 시 생략 가능 |

```
입력: outline.md (스프링부트 웹 개발) + seed.md
    ↓
1단계: 도메인 표준 커리큘럼 도출
  - 일반적으로 Spring Boot 입문서가 다루는 주제 리스트 도출

2단계: 현재 목차와 비교
  - 있는 것: Controller, JPA, 인증, 배포
  - 없는 것: 의존성 주입, 예외 처리, 테스트, 설정 관리

3단계: seed.md 의도 필터 적용 + 우선순위 분류
  [필수] 의존성 주입 — 이해 없이 Spring을 쓸 수 없음
  [권장] 예외 처리 — 실무 차별점, "실수→해결" 소재로 좋음
  [선택] 테스트 — 분량 고려 시 부록 처리 가능
  [선택] 설정 관리 — seed.md 의도(이해) 범위 초과, 생략 권장

※ 저자 승인 후에만 반영
```

### C. 글쓰기 스킬 (5개)

텍스트를 생성하는 스킬. 책뿐 아니라 블로그, 문서, 발표 자료 등에 재사용 가능.

---

#### C1. 비유-생성기

> 기술 용어를 일상 비유로 변환한다.

| 항목 | 내용 |
|------|------|
| **입력** | 기술 용어 + 맥락 |
| **출력** | 비유 (길이 제한 없음) |
| **규칙** | 독자가 100% 아는 일상적인 것만. 기술 용어로 비유 금지. 길이 제한 없음 |

```

#### D1. 용어-탐지기

| **안 하는 일** | 대체안 제안 (그건 C1 비유-생성기의 일) |

#### D2. 톤-검사기

> 텍스트가 대화체/스토리텔링 톤을 유지하는지 검사한다.

| 항목 | 내용 |
|------|------|
| **입력** | 텍스트 |
| **출력** | 톤 위반 목록 + 위치 |
| **검사 항목** | 딱딱한 정의 문장, 교과서 어투, 수동태, 독자와 대화 없는 구간 |

```
입력: Ch.3 이야기 파트
출력:
  ⚠️ 8줄: "세션이란 서버 측에서 관리하는 상태 정보이다"
     → 정의 문장. 이야기 파트에서는 비유로 풀어야 함
  ⚠️ 15~22줄: 7문장 연속 독자 호출 없음
     → "여기서 한 가지 궁금하지 않나요?" 같은 문장 삽입 권장
  ✅ 3줄: "혹시 이런 경험 있으신가요?" → OK
```

#### D3. 파트-분리-검증기

```

  ✅ 용어 정리: 기술 파트 첫 머리에 있음

## 코드 워크플로우와의 관계

```
┌─────────────────────────┐
│    코드 워크플로우         │
│    (별도 설계)            │
│                           │
│  완성 코드를 만드는 단계    │
│  (코드 작성, 문법, 컨벤션)  │
│                           │
│  산출물: 완성 코드          │
└────────────┬──────────────┘
             │
             ▼ (완성 코드 전달)
┌─────────────────────────┐
│    집필 워크플로우         │
│    (이 문서 + 워크플로우-  │
│     설계-v2.md)           │
│                           │
│  완성 코드를 분석하고       │
│  책을 쓰는 단계            │
│                           │
│  STEP 2 코드분해-에이전트가 │
│  A1~A5 스킬로 분석 수행     │
└─────────────────────────┘
```

코드 워크플로우가 만든 **완성 코드**를 집필 워크플로우의 **코드분해-에이전트**가 분석 스킬(A1~A5)로 해부한다.
코드를 작성하는 스킬(문법, 컨벤션, 빌드 등)은 코드 워크플로우에만 존재하며 이 문서에 포함하지 않는다.
