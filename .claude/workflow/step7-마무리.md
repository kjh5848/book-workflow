# STEP 7. 마무리 — "서문, 맺음말"

> **목적**: 책의 처음과 끝을 닫는다.
> **에이전트**: writer → editor → publisher
> **검토**: [감수 검토]
> **산출물**: `book/front/preface.md`, `book/back/afterword.md`

---

## 서문 작성 규칙

서문은 **저자의 경험담이 아니라, 독자에게 "왜 배워야 하는가"를 알려주는 글**이다.

### 원칙

1. **저자 경험 금지**: "내가 3년간 삽질했다" 식의 저자 이야기를 넣지 않는다. 프롤로그가 그 역할을 한다.
2. **독자에게 말하듯이**: 존댓말 대화체로, 독자가 처한 현실을 짚어주며 시작한다.
3. **기술별 현실적 필요성**: 각 기술이 왜 지금 필요한지를 현실 상황(채용 공고, 실무 문제, 시장 변화)으로 설명한다.
4. **문제 → 기술 → 의미**: 각 기술 문단은 "이런 문제가 생긴다 → 이 기술이 해결한다 → 그래서 알아둘 필요가 있다" 흐름을 따른다.

### 구조

```
1. 훅 (독자의 현재 상황을 짚는 1~2문단)
   — "CRUD는 만들 수 있는데 그 다음이 막막하다"
2. 기술별 필요성 (기술 수만큼 문단)
   — 각 문단: 현실 문제 → 기술 이름 → 왜 필요한가
   — 기술 간 자연스러운 흐름 연결
3. 마무리 (책의 목표 + 읽는 방법)
   — 이 책이 무엇이고 무엇이 아닌지
   — 대상 독자 + 읽는 방법
```

### 금지

- 저자 경험/동기 ("이 책을 쓰게 된 계기는...")
- 설교 ("반드시 알아야 합니다")
- 시간 의존 표현 ("최근에 나온", "2026년 현재")
- 기술 나열만 하는 목록형 ("1장은 Docker, 2장은 OAuth...")
- 감사 인사 (서문이 아닌 별도 페이지로)

### 좋은 패턴

- 채용 공고, 서버 디스크 알림, 로그인 풀림 등 **독자가 겪을 법한 구체적 상황**으로 기술을 도입
- "선택이 아니라 기본 인프라가 된 지 오래입니다" — 과장 없이 현재 위상을 한 문장으로
- "서버 디스크가 가득 찼을 때 자연스럽게 찾게 되는 해결책" — 기술을 특별한 것이 아니라 자연스러운 다음 단계로 포지셔닝

## 맺음말 작성 규칙

맺음말은 **프롤로그의 대칭**이다. 프롤로그가 "첫 티켓 앞에 선 순간"이라면, 맺음말은 "아홉 장을 넘긴 후의 시점"이다.

### 원칙

1. **질문 없이 작성**: seed.md + 프롤로그 + 챕터 흐름에서 자연스럽게 도출한다.
2. **프롤로그와 호응**: 프롤로그에서 던진 이미지(사수의 티켓, 터미널 앞에서 멈춘 순간 등)를 회수한다.
3. **독자의 다음 단계**: 이 책이 끝이 아니라 출발점임을 전달한다. 설교 없이.
4. **짧게**: 서문보다 짧다. 1~2페이지.

### 구조

```
1. 회수 (프롤로그의 이미지를 되돌아보는 1~2문단)
2. 지금 독자의 위치 (아홉 가지를 훑어본 지금, 뭐가 달라졌나)
3. 다음 방향 (깊이 파고 싶다면 어디로 가면 되는지)
4. 닫기 (한두 문장으로 마무리)
```

### 금지

- 감사 인사 (별도 페이지)
- 기술 요약 반복 (서문에서 이미 했다)
- "이 책이 도움이 되었으면 좋겠습니다" 류의 클리셰

## 부록 없음

**이 워크플로우는 부록을 만들지 않는다.** 실습 경계 원칙(style.md #11)에 따라, 실습 코드에 등장하지 않는 심화 내용은 챕터에도 넣지 않는다.

- 실습에 등장하는 설정/캡처 → 해당 챕터 기술 파트 내 실습 흐름에 포함
- 실습에 없는 심화 → "이런 게 있다" 한 줄 언급만 하고 넘어간다
- outline에서 "부록으로 이동"이라고 표시된 항목 → 실습에 없으면 삭제

## 흐름

```
1. 서문 작성 (seed.md + outline 기반)
2. 맺음말 작성 (프롤로그 + 챕터 흐름 기반)
3. 최종 제목 + 부제목 확정
4. 전체 분량 확인
    ↓
[감수 검토]
  - 독자 대변인: 맺음말이 독자에게 용기를 주나?
  - 이야기 편집장: 서문이 진정성 있나?
    ↓
저자 확인 → 최종 산출물 생성
```

> **전체 STEP 7은 질문 없이 진행한다.** seed.md, 프롤로그, outline, 챕터 본문이 충분한 입력이다.
