# 집필 워크플로우 v3 — 사용 가이드

## 컨셉

**"기술의 대가가, 이 기술을 처음 접하는 사람에게 이야기로 알려주는 책"**

- 스토리텔링 기반. 교과서가 아니라 이야기.
- 어려운 용어는 비유로 풀어 설명.
- 완성 코드를 버전별로 쪼개서 점진적으로 성장하는 구조.
- 하나의 AI(Claude)가 처음부터 끝까지 관통.

## 시작하기

Claude Code에서 이 프로젝트 폴더를 열고 `새 책 만들기`를 입력하세요.

## 전체 워크플로우 (7 STEP)

```
Phase 1 ── 의도 확립
  STEP 1. 씨앗              "이 책은 뭐다"

Phase 2 ── 재료 파악
  STEP 2. 코드 해부          "재료가 뭐가 있지"

Phase 3 ── 이야기 설계
  STEP 3. 시나리오 + 버전     "어떤 순서로 이야기하지"
  STEP 4. 뼈대 세우기         "목차와 코드 실습 배치"

Phase 4 ── 집필
  STEP 5. 챕터 집필 (반복)    "쓴다"

Phase 5 ── 완성
  STEP 6. 프롤로그 + 로드맵   "숲을 보여준다"
  STEP 7. 마무리              "서문, 맺음말"
```

## 사용 가능한 명령어

| 명령어 | 설명 |
|--------|------|
| `새 책 만들기` | 새 프로젝트 시작 → STEP 1부터 진행 |
| `현재 상태` | 진행 상황 확인 |
| `씨앗 심기` | STEP 1 진행 |
| `코드 분석` | STEP 2 진행 |
| `시나리오 설계` | STEP 3 진행 |
| `뼈대 세우기` | STEP 4 진행 |
| `챕터 작성 [번호]` | STEP 5 해당 챕터 진행 |
| `검토 [챕터]` | 의도감시 + 감수 검토 재실행 |
| `프롤로그 생성` | STEP 6 진행 |
| `마무리` | STEP 7 진행 |

## 워크플로우 파일 구조

```
.claude/workflow/
├── README.md                    # 이 파일
├── step1-씨앗.md                # STEP 1: 의도 확립
├── step2-코드해부.md             # STEP 2: 코드 분석
├── step3-시나리오.md             # STEP 3: 시나리오 + 버전 설계
├── step4-뼈대.md                # STEP 4: 목차 + 코드 실습 배치
├── step5-챕터집필.md             # STEP 5: 이야기 파트 + 기술 파트
├── step6-프롤로그.md             # STEP 6: 프롤로그 + 로드맵
├── step7-마무리.md              # STEP 7: 서문, 맺음말
├── review-guide.md              # 검토 모드 가이드 (인사이트/의도감시/감수)
└── legacy/                      # v3 이전 워크플로우 파일
```

## 프로젝트 폴더 구조

```
projects/{책이름}/
├── answers.md                  ← 모든 답변 누적
├── planning/                   ← STEP 1~4 산출물
│   ├── seed-v1.md
│   ├── code-analysis-v1.md
│   ├── scenario-v1.md
│   └── outline-v1.md
├── chapters/                   ← STEP 5 산출물
├── book/
│   ├── front/                  ← 프롤로그, 로드맵, 서문
│   ├── body/                   ← 본문 챕터
│   └── back/                   ← 맺음말
├── versions/                   ← 버전별 예제 코드
├── questions/
│   ├── pending/
│   └── done/
├── code/                       ← 완성 코드
└── review/
    └── feedback-log.md
```

## 핵심 규칙

- **상수는 질문하지 않는다**: 톤(대화체), 스타일(스토리텔링), 챕터 구조(이야기→기술), 코드 분류(실습/설명/참고)
- **변수는 인사이트 질문으로 채운다**: 단순히 "뭐야?"가 아니라 "이건 생각해봤어?"
- **의도가 필터다**: seed.md의 의도가 이후 모든 결정의 기준
- **실패 먼저**: 정답부터 알려주지 않는다. 실수 → 에러 → 해결 순서
