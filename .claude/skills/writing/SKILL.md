# 집필 스킬

## 로드 시점

| STEP | 상황 |
|------|------|
| STEP 5. 챕터 집필 | 이야기 파트 + 기술 파트 작성 |
| STEP 6. 프롤로그 | 전체 개념을 이야기로 엮기 |
| STEP 7. 마무리 | 서문, 에필로그 작성 |

## 핵심 규칙

- 톤: **대화체** — 독자에게 친구처럼 말하기
- 스타일: **스토리텔링** — 모든 챕터는 하나의 에피소드
- 구조: **이야기 파트(코드 없음) → 기술 파트(코드 + 정의)**
- 비유: 일상적인 것만 사용. **기술 용어로 비유 금지**
- 코드: 이야기 파트에 코드 없음. 기술 파트에서만
- 닫기: 모든 챕터는 "이것만은 기억하자" + 다음 예고로 마무리

## 상수 (자동 적용, 질문하지 않음)

| 상수 | 값 |
|------|-----|
| 포맷 | 짧은 쿡북 (100p 권장, 초과 허용) |
| 스타일 | 스토리텔링 |
| 톤 | 대화체 (전체 존댓말) |
| 언어 수준 | 쉽게 (비유 필수) |
| 비유 길이 | 제한 없음 |
| 챕터 내부 구조 | 이야기 파트 → 기술 파트 |
| 코드 분류 | 실습 / 설명 / 참고 3단계 |
| 성장 방식 | 버전별 성장 (챕터 = 한 버전) |
| 코드 성격 | 교육용 (실무 코드 금지) |
| 용어 정리 | 비유 → 기술 파트에서 정식 정의 필수 |
| 독자 수준 | 배경지식 있음, 이 책의 주제만 모름 |
| 책 유형 | 개념서 |
| 코드 비중 | 낮음 (이야기 파트 없음, 기술 파트에서만) |
