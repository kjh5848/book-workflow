# 버전별 점진적 성장 패턴

> 출처: v1 writing-concept/project-buildup + v3 적응
> 관련 스킬: B2.스냅샷-설계기

---

## 원칙

하나의 프로젝트를 처음부터 끝까지 버전별로 만들어간다.
각 챕터는 이전 챕터의 결과물 위에 기능을 추가한다.

---

## 챕터 도입 패턴

```markdown
### 여기까지 왔다
- [x] v0.1: Hello World
- [x] v0.2: 글 목록 페이지
- [ ] **v0.3: 데이터 저장** ← 이번 챕터

지난 챕터에서 (이전 성과)를 완성했다.
이번 챕터에서는 (새 기능)을 추가한다.
```

---

## 코드 확장 표시

기존 코드와 새로 추가된 코드를 명확히 구분한다:

```
기존 코드 (Ch.2에서 작성, 변경 없음)
─────────────────────────────
새로 추가된 코드 (Ch.3)
```

---

## 챕터 닫기 패턴

```markdown
### 이것만은 기억하자
- (이번 챕터 핵심 한 줄)
- (새 기능)이 추가되었다. 다음 챕터에서는 (다음 기능)을 만든다.
```

---

## 누적 구조 다이어그램

각 챕터 시작에 **지금까지의 전체 구조**를 보여준다.
새로 추가되는 부분을 강조(하이라이트)한다.

```
v0.1: [Controller]
v0.2: [Controller] → [Service]
v0.3: [Controller] → [Service] → [Repository] ← NEW
```

---

## 독립 실행 원칙

각 버전(versions/v0.X/)은 독립적으로 실행 가능해야 한다.
이전 챕터를 완료하지 않아도 해당 버전의 코드를 바로 실행할 수 있어야 한다.
