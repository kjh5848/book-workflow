# 박스 스타일 가이드

> 출처: v1 writing/box-style
> 관련 스킬: D4.포맷-검증기

---

## 박스 종류

| 박스 | 용도 | 마크다운 |
|------|------|---------|
| **팁** | 유용한 정보, 꿀팁 | `> **팁: 제목**` |
| **주의** | 흔한 실수, 주의사항 | `> **주의: 제목**` |
| **경고** | 데이터 손실, 심각한 에러 | `> **경고: 제목**` |
| **참고** | 부가 설명, 레퍼런스 | `> **참고: 제목**` |
| **질문** | Q&A 형식 | `> **질문: 질문?**` |

---

## 사용 규칙

1. 한 섹션에 연속 박스 **최대 2개**
2. **경고** 박스는 진짜 심각한 경우에만 (데이터 손실, 서버 다운 등)
3. 박스 안에서도 대화체 유지
4. 박스 안 코드 블록은 들여쓰기 주의

---

## 예시

```markdown
> **팁: 세션 타임아웃 설정**
> 개발할 때는 타임아웃을 길게 잡아두면 편해요.
> 배포할 때는 반드시 줄여야 합니다.

> **주의: .env 파일을 Git에 올리지 마세요**
> API 키가 포함된 .env 파일은 반드시 .gitignore에 추가하세요.
> 한번 올라가면 히스토리에 남아서 삭제해도 소용없습니다.

> **질문: 세션과 쿠키는 같은 건가요?**
> 아닙니다! 쿠키는 브라우저에 저장되는 작은 메모지고,
> 세션은 서버에서 관리하는 출입 기록입니다.
```
