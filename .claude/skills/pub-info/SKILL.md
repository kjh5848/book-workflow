# 출판정보 생성 스킬

## 로드 시점

PDF 빌드 완료 후, `출판정보 생성` 명령으로 실행.
publisher 에이전트가 PDF 빌드 파이프라인 완료 후 자동 호출.

## 호출 방식

```
출판정보 생성          → POD + 전자책 두 벌 생성
출판정보 생성 POD      → B5(188x257mm)만
출판정보 생성 전자책    → A4(210x297mm)만
```

## 판형 상수

| 유형 | 판형명 | 크기(mm) |
|------|--------|----------|
| POD | 4x6배판(B5) | 188 x 257 |
| 전자책 | 국배판(A4) | 210 x 297 |

## 자동 추출 소스

| 항목 | 소스 | 추출 방법 |
|------|------|----------|
| 책제목 | `planning/seed-v*.md` | `## 주제` 섹션 첫 줄 |
| 부제목 | `planning/seed-v*.md` | 주제 줄에서 `—` 뒤 부분 |
| 목차 | `chapters/*.md` | h1~h3 헤딩 추출 |
| 페이지수 | `book/output/*.pdf` | PDF 페이지 카운트 (`python3 -c "..."`) |
| 책소개 | `planning/seed-v*.md` + `chapters/*.md` | 아래 "책소개 작성 규칙" 참조 |
| 키워드 | `planning/seed-v*.md` | `## 핵심 메시지` + 각 챕터 핵심 개념 |

| 본문언어 | 고정 | 한국어 |

## 실행 흐름

```
1. seed-v*.md 최신 버전 읽기 → 책제목, 부제목, 책소개, 키워드 추출
2. chapters/*.md h1~h3 추출 → 목차 생성
3. book/output/*.pdf 페이지 카운트 → 페이지수
   (PDF 없으면 outline 분량 합산으로 대체)
4. prologue.md 또는 chapters/01-*.md → 요약/본문일부 추출
5. 판형 상수 적용 (POD/전자책)
6. 유저에게 입력 항목 질문 (시리즈, 저자명, 제본형태, 출판사 정보 등)
7. 템플릿에 채워서 markdown 생성
8. 판권지 테이블 자동 생성
```

## 산출물

## 요약/본문일부 작성 규칙

요약/본문일부는 책소개와 다른 역할이다. 책소개가 "왜 사야 하는가" 설득이라면, 요약/본문일부는 "이 책의 글이 이런 느낌이다" 체험이다.

### 선택 기준

1. **책의 톤을 대표하는 구간**: 이 구절만 읽으면 책 전체의 문체와 분위기를 알 수 있어야 한다
2. **독립적으로 읽혀야 한다**: 앞뒤 맥락 없이도 이해 가능한 구간
3. **읽고 나면 더 읽고 싶어야 한다**: 결론을 주지 않고 호기심만 남기는 구간
4. **200~500단어**: 짧은 발췌 권장 (서점 등록용)

### 추천 소스 (우선순위)

1. 프롤로그 오프닝 (역순 훅 장면) — 독립적이고 톤 대표
2. CH01 이야기 파트 도입부 — 첫 문제 상황
3. 시나리오 전환점 장면 — 극적 변화

### 금지

- 기술 파트 코드 발췌
- 책소개와 동일한 내용 복붙
- 결론/해결을 보여주는 구간 (스포일러)

## 책소개 작성 규칙

책소개 = 엘리베이터 피치. 30초 안에 "이 책을 왜 사야 하는지" 전달해야 한다.

### 4단 구성

```
1. 훅 (1~2문장)
   "이 책은 ~하며 ~한다!"
   대상 독자 + 핵심 기술 + 만드는 것을 한 문장으로 정의

2. 구성 상세 (1~2문단)
   단계별 또는 파트별로 뭘 하는지 나열
   구체적 프로젝트명/기술명 직접 등장 (추상적 설명 금지)

3. 배울 내용 (불릿 5~10개)
   "이 책을 통해 배우는 내용은 다음과 같다" 또는
   "이 책을 통해 이런 걸 할 수 있게 된다"
   종결: "~할 수 있다" / "~해본다" / "~하는 방법"

4. 대상 독자 (불릿 3~6개, 선택)
   "이런 분께 추천한다"
   구체적 상황으로 서술 ("AWS 배포를 처음 접하는 분")
```

### 문체

- 능동형 종결: "~한다", "~할 수 있다", "~해본다"
- 구체적 기술명/프로젝트명 반복 등장
- 추상적 표현 금지: "다양한 기술을 배울 수 있습니다" → 기술명 직접 나열
- 짧은 문장, 짧은 문단 (스킵해도 핵심이 보이게)

### 소스

seed-v*.md의 `## 책의 의도` + `## 차별점` + chapters/*.md h1 제목에서 기술명 추출

## 목차 추출 규칙

- `chapters/*.md` 파일을 번호순으로 읽는다
- `#` (h1), `##` (h2), `###` (h3) 헤딩만 추출
- `## 이야기 파트`, `## 기술 파트` 등 파트 구분 h2는 제외
- `<!-- -->` 주석 내부의 헤딩은 제외
- 들여쓰기로 계층 표현

```
Ch.1: Hallucination과 RAG
  이야기 파트
    1.1 입사 3일 차, 첫 번째 임무
    1.2 LLM의 자신감 넘치는 거짓말
    ...
```

## 페이지 카운트

```bash
python3 -c "
import subprocess
result = subprocess.run(['typst', 'query', 'book.typ', '<page>', '--field', 'number'], capture_output=True, text=True)
# 또는 PyPDF2/pikepdf로 PDF 페이지수 확인
"
```

PDF가 없으면 `outline-v*.md`의 `예상 분량` 합산값을 `(추정)` 표시와 함께 사용.
