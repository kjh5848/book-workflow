# 이미지 분석 가이드

## 교육적 의도 파악 기준

이미지를 분석할 때 다음 질문에 답한다.

1. **이 이미지가 없으면 독자가 무엇을 이해하지 못하는가?**
   - 개념의 관계 → 아키텍처
   - 시간 순서 → 프로세스
   - 추상적 개념 → 비유
   - 차이점 → 비교
   - 동작 결과 → 결과

2. **문맥에서 이 이미지의 역할은 무엇인가?**
   - 앞 문단이 "~처럼" "마치~" 등 비유를 쓰고 있다면 → 비유 이미지
   - 앞 문단이 시스템 구조를 설명하고 있다면 → 아키텍처
   - 앞 문단이 "먼저~", "그다음~" 등 순서를 설명하고 있다면 → 프로세스

3. **독자가 이 이미지에서 가장 먼저 봐야 할 것은?**
   - 이것이 프롬프트의 시각적 초점(focal point)이 된다

## 저작권 회피 전략

참고 이미지를 교육용으로 재해석할 때 준수할 원칙.

### 절대 복사 금지

- 원본의 정확한 레이아웃, 색상 배치, 아이콘을 복제하지 않음
- 원본의 텍스트를 그대로 사용하지 않음 (한국어로 변환)
- 원본의 브랜드 로고나 고유 디자인 요소를 포함하지 않음

### 재해석 방법

1. **핵심 교육 메시지 추출**: 이 이미지가 전달하려는 핵심 개념을 1문장으로 정리
2. **요소 추상화**: 구체적 UI/제품 → 범용 아이콘(심볼 패턴 사용)
3. **스타일 변환**: 원본 스타일 무시 → 프로젝트 미니멀 B&W 라인아트
4. **구도 재배치**: 같은 정보를 다른 배치로 표현 (좌우→상하, 원형→직선 등)
5. **라벨 한국어화**: 모든 텍스트를 한국어로

### 프롬프트 품질 체크리스트

- [ ] 베이스 스타일(16:9, B&W, 라인아트)이 포함되어 있는가
- [ ] 핵심 요소가 명확하게 기술되어 있는가
- [ ] 라벨이 한국어인가
- [ ] 원본과 시각적으로 구별되는 구도인가
- [ ] 공통 심볼 패턴을 활용하고 있는가
- [ ] Style 태그가 적절한가

## 문맥 읽기 범위

```
--- 위 3문단 (이미지가 어떤 맥락에서 등장하는지) ---

![캡션](이미지경로)        <-- 분석 대상
*그림 N-M: 캡션*

--- 아래 2문단 (이미지 이후 어떤 설명이 이어지는지) ---
```

문단 구분: 빈 줄(`\n\n`)을 기준으로 한다.
heading(`#`, `##` 등)도 문단으로 취급한다.
