# 검토 스킬

## 로드 시점

| STEP | 발동 검토 모드 |
|------|--------------|
| STEP 1. 씨앗 | 인사이트 + 감수 |
| STEP 2. 코드 해부 | 인사이트 + 감수 |
| STEP 3. 시나리오+버전 | 인사이트 + 감수 |
| STEP 4. 뼈대 | 인사이트 + 감수 |
| STEP 5. 챕터 집필 | 인사이트 + **의도감시** + 감수 (3개 모두) |
| STEP 6. 프롤로그 | 감수 |
| STEP 7. 마무리 | 감수 |

## 핵심 규칙

- 판정 3종: **PASS / CONDITIONAL_PASS / FAIL**
- FAIL: **실패 항목 + 수정 제안 필수**
- **최대 2회 재시도**, 이후 저자와 상의
- 피드백은 `review/feedback-log.md`에 기록
- STEP별로 발동하는 검토 모드 조합이 다름

## 담당 스킬

### Tier 1 (상세 규칙 필요)

| 스킬 | 역할 | 핵심 규칙 |
|------|------|----------|
| D3. 파트-분리-검증기 | 이야기/기술 파트 분리 검증 | 이야기 파트에 코드 블록 금지 |
| D4. 포맷-검증기 | 상수 준수 검증 | 구조, 코드 성격, 톤 체크 |
| D5. 의도-대조기 | seed.md 의도와 대조 | 범위 이탈, 깊이 초과 감지 |

### Tier 3 (자연 수행)

| 스킬 | 역할 |
|------|------|
| D1. 용어-탐지기 | 비유 없이 등장하는 어려운 전문 용어 탐지 |

## 3개 검토 모드

| 모드 | 시점 | 내용 |
|------|------|------|
| **인사이트 검토** | STEP 1~5 | 저자가 놓친 부분을 짚어주는 추가 질문 |
| **의도감시 검토** | STEP 5만 | seed.md 의도와의 정합성 7항목 검증 |
| **감수 검토** | 전 STEP | 3인 편집장(기술/독자/이야기) 각 2~3개 질문 |
