# 코드 스킬

## 로드 시점

| STEP | 상황 |
|------|------|
| STEP 2. 코드 해부 | 완성 코드 구조/기능/기술스택/의존성 분석 |
| STEP 4. 뼈대 | 코드 파일 태깅 + 파일 계층 시각화 |
| STEP 5. 챕터 집필 | 기술 파트 코드 설명 작성 |

## 핵심 규칙

- 코드 블록은 **기술 파트에서만** (이야기 파트에 절대 없음)
- 모든 코드 블록: **목적 한 줄 → 코드 → 설명 → 실행 결과**
- 핵심만 보여줌 (10~15줄), 전체 코드는 versions/ 참조
- **언어 태그 필수**
- 쉘/bash 블록은 상세 설명 불필요
- **교육용 코드만** (실무 코드 금지)

## 담당 스킬

### Tier 1 (상세 규칙 필요)

| 스킬 | 역할 | 핵심 규칙 |
|------|------|----------|
| B3. 코드-태거 | 파일에 [실습/설명/참고] 태그 | [실습]=핵심(직접 작성), [설명]=읽기, [참고]=파일명만 |

### Tier 2 (입출력 정의)

| 스킬 | 입력 → 출력 |
|------|------------|
| A1. 구조-스캐너 | 프로젝트 경로 → 디렉토리 트리 (파일명 + 한 줄 설명) |
| A2. 기능-추출기 | 코드 파일 → 기능 목록 (기능명 + 관련 파일 + 한 줄 설명) |
| A3. 기술스택-탐지기 | 빌드 파일 + 코드 → 기술 스택 (기술명 + 버전 + 용도) |
| A4. 의존성-매퍼 | 코드 파일 → 의존성 그래프 (A→B: "A가 B를 사용") |
| A5. diff-생성기 | 버전A vs 버전B → 신규/변경/삭제 + 핵심 변경 요약 |
| B4. 계층-생성기 | 태그 붙은 파일 목록 → 트리 시각화 (태그 + 한 줄 설명) |
