# 갭 분석 절차

> 출처: v1 planning/gap-analysis + v3 적응
> 관련 스킬: B6.갭-분석기

---

## 우선순위 분류

| 등급 | 정의 | 예시 |
|------|------|------|
| **[필수]** | 이거 없으면 독자가 핵심 내용을 이해할 수 없음 | HTTP 기초 없이 REST API 설명 |
| **[권장]** | 있으면 책의 차별점이 됨, 독자 만족도 상승 | 에러 핸들링 베스트 프랙티스 |
| **[선택]** | 심화/부록 처리 가능, 분량 고려 시 생략 가능 | 성능 최적화, 배포 자동화 |

---

## 절차

1. **도메인 표준 커리큘럼 도출**
   - "이 주제의 입문서가 보통 다루는 것"을 도출
   - 필요시 웹 검색으로 확인

2. **현재 목차와 비교**
   - outline.md에 있는 것 vs 표준에 있는 것

3. **의도 필터 적용**
   - seed.md의 "의도 밖 범위"에 해당하면 제외
   - 남은 것에 [필수/권장/선택] 우선순위 분류

4. **저자 승인**
   - 분석 결과를 저자에게 보여주고 반영 여부 결정
   - **저자 승인 없이 목차에 반영하지 않는다**

---

## 공통 갭 체크 항목

| 항목 | 설명 |
|------|------|
| 기본 개념 | 핵심 용어 정의가 빠져있지 않은가? |
| 비교 분석 | 유사 기술과의 비교가 필요하지 않은가? |
| 트러블슈팅 | 흔한 에러/실수 대응이 빠져있지 않은가? |
| 실용 패턴 | 실제 적용 시 필요한 패턴이 있는가? |
| 평가/검증 | 잘 동작하는지 확인하는 방법이 있는가? |

---

## 출력 형식

```markdown
## 갭 분석 결과

### [필수]
- **HTTP 기초**: REST API 챕터 전에 필요. → Ch.2에 선행 개념으로 추가 제안

### [권장]
- **에러 핸들링**: 실습 시 독자가 만날 에러 대응. → 각 챕터 시행착오에 포함 제안

### [선택]
- **배포 자동화**: 의도(이해)와 거리 있음. → 부록 또는 생략 제안
```
