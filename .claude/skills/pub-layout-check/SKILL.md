# layout-check — 레이아웃 분석

## 스크립트

- **원본**: `references/scripts/pdf_layout_checker.py` (스킬 소유)
- **프로젝트 참조**: `book/pdf_layout_checker.py` (심볼릭 링크 → 스킬)

## 역할

빌드된 PDF를 페이지별로 분석하여 레이아웃 문제를 자동 감지하고, 심각도별 피드백을 제공한다.

## 감지 항목

| 이슈 | 심각도 | 조건 | 제안 |
|------|--------|------|------|
| **빈 페이지** | high | 텍스트·이미지 블록 없음 | pagebreak 제거 |
| **고아 콘텐츠** | high | ≤4줄 + 하단 50%+ 빈 공간 | 이전 페이지로 당기기 |
| **낮은 사용률** | medium | 콘텐츠 45% 미만 | 이미지/코드가 밀린 패턴 |
| **과대 이미지** | medium | 이미지가 페이지 70%+ | max-width 축소 |
| **밀림 패턴** | medium | 이전 페이지 55% 미만 + 다음 페이지 이미지 | auto-image 자동 축소 |

## 출력 형태

### 페이지 사용률 시각화

```
p03 |#################.......................|  45%  <<빈 공간>>
p04 |#################################.......|  85%
p05 |###############################.........|  79%
p06 |########................................|  22%  <<<고아>>>
```

### 이슈 리포트

```
[!] 페이지 23: 2줄만 있고 하단 96% 빈 공간
    -> 이전 페이지에서 약간만 줄이면 이 줄들이 이전 페이지에 들어갈 수 있음
[~] 페이지 6~7: 이전 페이지 26% 사용 후 다음 페이지에 이미지 시작
    -> 이미지가 이전 페이지에 안 들어가서 밀린 패턴
```

## 실행

```bash
# 독립 실행
.pdf_venv/bin/python3 book/pdf_layout_checker.py book/output.pdf

# 빌드 파이프라인 내 자동 실행 (build 스킬의 6/6 단계)
```
