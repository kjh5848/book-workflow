# page-fit — 페이지 밀도 조정

## 역할

layout-check 스킬이 감지한 이슈를 해결하는 전략을 제공한다.
자동으로 적용할 수 있는 수정과 수동 판단이 필요한 수정을 구분한다.

## 이슈별 해결 전략

### 1. 고아 콘텐츠 (1~3줄 넘어감)

**자동 수정 가능한 경우:**
- 이전 페이지의 이미지 max-width를 5~10% 줄여 공간 확보
- 이전 페이지의 수평선(`---`) 제거로 여백 확보

**수동 판단 필요:**
- 이전 섹션의 텍스트를 약간 줄이기 (1~2문장 삭제/축약)
- 다음 섹션이 h1이면: 고아 줄을 허용 (챕터는 무조건 새 페이지)
- 다음 섹션이 h2/h3이면: pagebreak 없이 이어붙이기 가능

### 2. 이미지 밀림 (이전 페이지 반 이상 빈 공간)

**자동 수정:**
- auto-image가 이미 `layout()` 기반 자동 축소 시도
- 이미지의 max-width를 단계적으로 줄여 재빌드:
  `0.7 → 0.6 → 0.5 → 0.4`

**수동 판단:**
- 이미지 전 텍스트를 추가하여 이미지 위치를 조정
- 이미지를 섹션 끝으로 이동

### 3. 빈 페이지

**자동 수정:**
- heading show rule의 `pagebreak(weak: true)` 제거 (해당 heading에만)
- 또는 이전 콘텐츠의 여백 축소

**수동 판단:**
- 의도적 빈 페이지인지 확인 (Part 구분 등)

### 4. 과대 이미지

**자동 수정:**
- `_detect_image_max_width()`에서 해당 유형의 max-width 축소
- Mermaid는 종횡비 자동 감지 (가로형 0.75 / 중간 0.55 / 세로형 0.45)
- autocrop으로 불필요한 공백 먼저 제거

### 5. 이야기 파트 밀도 부족 → 2단 레이아웃

**적용 기준:**
- `== 이야기 파트` 내부, 이미지·코드·표 없이 텍스트만 4단락 이상 연속
- 해당 구간 페이지 사용률 70% 미만
- 기술 파트, 이미지/코드/표/인용 포함 구간에는 적용 금지

**수동 판단 필요** — 상세 규칙은 [fit-strategies.md](references/fit-strategies.md) 전략 8 참조.

## 자동 수정 워크플로우

```
1. layout-check 실행 → 이슈 목록
2. 이슈별 자동 수정 가능 여부 판단
3. 자동 수정 적용 (이미지 크기, 수평선 제거 등)
4. 재빌드
5. 재분석 → 이슈가 줄었는지 확인
6. 남은 이슈는 사용자에게 보고
```
