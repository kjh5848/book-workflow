# 페이지 맞춤 전략 상세

## 전략 1: 이미지 단계적 축소

이미지가 밀려서 빈 공간이 생길 때, max-width를 단계적으로 줄인다.

```
시도 1: 원래 max-width (예: 0.6)
시도 2: max-width - 0.1 (예: 0.5)
시도 3: max-width - 0.2 (예: 0.4)
```

build_pdf_typst.py의 `_detect_image_max_width()`를 수정하거나,
특정 이미지에 대한 오버라이드를 추가.

## 전략 2: 수평선 제거

h2/h3 직전의 `---`는 불필요한 여백을 만든다.
heading show rule이 이미 여백(v(20pt), v(16pt))을 생성하기 때문.

**규칙**: `fix_typst_content()`에서 heading 직전 `#horizontalrule` 제거:
```python
text = re.sub(r'#horizontalrule\n+(?==)', '', text)
```

## 전략 3: heading 페이지 넘김 조건부 적용

모든 h1에 `pagebreak(weak: true)`를 적용하면, 이전 섹션 마지막 줄이 넘어갔을 때 빈 페이지 문제 발생.

**해결 옵션:**
- 프롤로그 → CH01 같은 전환은 pagebreak 유지
- 같은 챕터 내 h1은 pagebreak 생략 (해당 없음, h1은 챕터 제목만)

## 전략 4: 코드 블록 앞 가이드 텍스트

긴 코드 블록(15줄+) 앞에 1~2줄 가이드 텍스트를 추가하면:
- 코드 블록의 시작 위치가 조금 내려감
- 이전 페이지의 빈 공간이 텍스트로 채워짐

예:
```markdown
다음 코드는 네 단계로 나뉩니다. 문서 로드, 분할, 임베딩, 저장 순서입니다.
```

## 전략 5: 콘텐츠 축약

고아줄 문제가 반복될 때, 해당 섹션의 텍스트를 약간 줄인다.

**기준:**
- 고아줄이 1줄이면: 이전 페이지에서 1문장 축약
- 고아줄이 2~3줄이면: 이전 페이지에서 2~3문장 축약
- 축약할 때 **의미가 변하지 않도록** 주의

## 전략 6: 여백 미세 조정

특정 페이지에서만 여백을 약간 줄이는 방법.
Typst에서는 `#set page(margin: ...)` 를 중간에 삽입할 수 있지만,
일관성을 해치므로 최후 수단으로만 사용.

## 전략 7: 연속 흐름 보장

인접 페이지에서 콘텐츠가 자연스럽게 이어져야 할 때 사용.

**감지 기준:**
- 페이지 사용률 < 50% + 다음 페이지에 같은 섹션 콘텐츠 연속
- 이미지가 밀려서 앞 페이지에 빈 공간이 생긴 경우 (push pattern)

**해결:**
1. Mermaid 다이어그램 종횡비 조정 — `flowchart TD` + `direction LR` subgraph로 가로 넓게
2. 이미지 max-width 축소 — 현재 페이지에 들어가도록
3. 앞 페이지 텍스트 축약 — 이미지를 현재 페이지로 끌어당기기

## 전략 8: 2단 레이아웃 (이야기 파트 한정)

이야기 파트에서 **텍스트만으로 이루어진 긴 구간**의 페이지 밀도를 높일 때 사용.

### 적용 조건 (모두 충족해야 함)

| 조건 | 설명 |
|------|------|
| **파트** | `== 이야기 파트` 내부에서만 적용 |
| **연속 텍스트** | 이미지·코드 블록·표 없이 텍스트만 4단락 이상 연속 |
| **밀도 부족** | 해당 구간이 포함된 페이지 사용률이 70% 미만 |
| **가독성** | 2단으로 나눴을 때 한 컬럼 너비가 최소 60mm 이상 |

### 적용 금지 조건

| 금지 조건 | 이유 |
|-----------|------|
| 기술 파트 | 코드 블록이 2단에서 잘려 가독성 저하 |
| 이미지 포함 구간 | 이미지와 텍스트가 교차하면 레이아웃 혼란 |
| 인용 블록(blockquote) 포함 | 좁은 컬럼에서 인용 블록이 지나치게 길어짐 |
| 표(table) 포함 | 표 너비가 컬럼에 맞지 않음 |
| 3단락 이하 짧은 구간 | 2단 전환/복원의 시각적 비용 대비 효과 없음 |

### Typst 구현

```typst
// 2단 시작 (이야기 파트 텍스트 구간)
#columns(2, gutter: 14pt)[
  텍스트 내용...
]
// 1단 복원 — 이미지, 코드 블록, 기술 파트 등

// 이미지가 중간에 나오면 2단을 닫고 이미지를 전체 너비로 배치
]
#auto-image("path.png", alt: [설명], max-width: 0.6)
#columns(2, gutter: 14pt)[
  이어지는 텍스트...
]
```

### 빌드 스크립트 적용 방식

`build_pdf_typst.py`의 후처리에서 자동 적용하려면:

1. `== 이야기 파트` ~ `== 기술 파트` 범위 감지
2. 해당 범위 내에서 이미지·코드·표·인용 없는 연속 텍스트 구간 찾기
3. 4단락 이상이면 `#columns(2, gutter: 14pt)[` ... `]`로 감싸기
4. 이미지가 중간에 나오면 2단을 닫고 → 이미지 → 다시 2단 열기

**현재 상태**: 규칙만 정의됨. 자동 적용 미구현. 수동 판단 후 마크다운에 직접 적용 가능.

## 우선순위

```
1순위: auto-image 자동 축소 (Typst 런타임)
2순위: autocrop 이미지 공백 제거 (빌드 전처리)
3순위: 수평선 제거 (빌드 후처리)
4순위: max-width 단계적 축소 (빌드 설정)
5순위: Mermaid 종횡비 자동 조절 (빌드 설정)
6순위: 연속 흐름 보장 (mermaid 방향, 이미지 조정)
7순위: 가이드 텍스트 추가 (마크다운 수정)
8순위: 2단 레이아웃 (이야기 파트, 수동 판단)
9순위: 콘텐츠 축약 (마크다운 수정)
10순위: 여백 미세 조정 (최후 수단)
```

1~5순위는 자동 적용, 6~10순위는 수동 판단 필요.
