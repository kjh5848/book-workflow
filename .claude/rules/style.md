# 글로벌 스타일 규칙

모든 에이전트가 준수하는 톤, 편집, 포맷 규칙.

---

## 톤

- 전체 존댓말 (-습니다/-입니다/-세요). 캐릭터 대화/독백만 구어체 유지
- 대화 표기. **역할**: "대사"
- 내면 독백. 괄호 `()` 사용 (이탤릭 금지)
- 등장인물. 역할명 사용 (**팀장**/**나**/**동료**), 조회 대상(A 사원, B 팀원). 실명 금지

## 편집

- **볼드** 양쪽 띄어쓰기 필수
- 문장 끝 콜론 금지 (`.` 사용)
- 능동태 우선 ("~되어진다" 금지)
- 영어 용어 처음 등장. 한국어(English) 형태. 이후 한국어만
- 이모지 전면 금지 (박스 스타일 포함 모든 곳에서 금지)
- 문맥 전환 시 빈 줄 삽입
- "## 이야기 파트" / "## 기술 파트" 대문 모두 제거. 전환 문장 + `---`로 대체
- 문단 구성. 같은 맥락의 문장은 빈 줄 없이 이어 쓴다. 한 줄짜리 문장이 연속으로 나열되지 않게 한다
- 숨돌리기 빈 줄. (1) 핵심 키워드/개념 도입 문장 전후 (2) 감정/반전 전환 지점 (3) 비유 마무리 직후에만 빈 줄 삽입. 단순 설명이 이어지는 구간은 문단으로 묶는다

## 금지 패턴

- 설교 ("이것이야말로...")
- 반복 강조 ("이것이 바로...이유")
- 정의 전 용어 사용
- 진부한 표현 ("두말할 필요 없이")
- 주관적 감탄 ("정말 놀라운 기능!")
- 시간 의존 표현 ("최근에 나온")
- 불필요한 감정 추임새 ("잠깐 기뻤습니다", "순간 설렜습니다" 등 서사에 기여하지 않는 한 줄짜리 감정 표현)

## 글쓰기 원칙

1. **이야기 먼저**: 모든 챕터는 이야기 파트로 시작한다.
2. **비유는 자유롭게**: 길이 제한 없음. 용어의 정식 설명이 길어지면 기술 파트로 넘긴다.
3. **기술은 뒤에서**: 이야기 파트가 끝난 후 기술 파트에서 상세 설명.
4. **비유 → 정의 2단계**: 어려운 개념은 반드시 2단계로 설명한다. 이야기 파트에서 비유로 먼저 풀고, 기술 파트에서 정식 정의를 내린다. 비유로 넘어간 용어는 기술 파트에서 반드시 정식 정의.
5. **한 챕터 한 버전**: 챕터가 끝나면 결과물이 한 단계 성장.
6. **점진적 깊이**: 쉬운 것 → 어려운 것.
7. **코드 최소화**: 이야기 파트에 코드 없음. 기술 파트에서만.
8. **챕터 닫기**: "이것만은 기억하자" + 다음 버전 예고.
9. **기술은 저자가 결정**: AI가 제안, 저자가 최종 결정.
10. **실패 먼저**: 정답부터 알려주지 않는다. 실수 → 에러 → 해결 순서.

## 출력 형식

- 모든 파일은 Markdown 형식으로 작성한다.
- 파일명은 한글 사용, 번호 접두사를 붙인다 (예: `01-시작하기.md`).
- 코드 블록에는 반드시 언어 태그를 명시한다.
- 이미지 위치는 `![설명](이미지경로)` 형태로 자리만 잡아두고, 추후 추가한다.

## 스킬 호출

- 챕터/프롤로그/에필로그 작성 시 → `writing` 스킬 호출
- 초안 완성 후 AI 패턴 교정 → `humanizer` 스킬 호출
- 스크린샷/캡처 작업 시 → `screenshot` 스킬 호출
