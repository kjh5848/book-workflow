# 브랜드 아키텍처

## 마스터 브랜드: 오픈스킬 (OpenSkill)

**오픈스킬** 은 모든 서비스의 상위 브랜드다. 하위 서비스는 서브도메인으로 분기하며, 각각 독립적인 역할을 가진다.

| 서브도메인 | 서비스 | 역할 |
|-----------|--------|------|
| `books.openskill.kr` | 오픈스킬 북스 | IT/AI 기술서적 출판, 판매 |
| `author.openskill.kr` | 오픈스킬 오서 | AI 집필 도구 (작가 대상 SaaS) |
| `ncs.openskill.kr` | 오픈스킬 NCS | 국가교육 기관 전용 프로그램 (B2G) |
| `edu.openskill.kr` | 오픈스킬 에듀 | AI 교육 에이전트 플랫폼 |

### 도메인 구조도

```
openskill.kr (마스터 브랜드)
  |
  +-- books.openskill.kr   (출판)
  +-- author.openskill.kr  (AI 집필 도구)
  +-- ncs.openskill.kr     (기관 전용 국가교육)
  +-- edu.openskill.kr     (AI 교육 에이전트)
```

## Branded House 모델 채택

브랜드 아키텍처에는 크게 4가지 모델이 있다.

| 모델 | 구조 | 예시 |
|------|------|------|
| **Branded House** | **마스터 브랜드 > 서비스** | **Google (Maps, Drive, Cloud)** |
| House of Brands | 모회사 > 독립 브랜드 | P&G (Tide, Pampers) |
| Endorsed | 하위 브랜드 by 마스터 | Marriott (Courtyard by Marriott) |
| Hybrid | 혼합 | Amazon (Prime, AWS) |

**오픈스킬에 Branded House 모델이 적합한 이유.**

1. 모든 서비스가 **openskill.kr** 서브도메인으로 통일된다. 사용자가 어떤 서비스를 쓰든 "오픈스킬"이라는 브랜드를 인식한다.
2. 출판(books), 교육(edu), 평가(ncs), 작가(author)는 모두 **"스킬을 열다"** 라는 하나의 미션 아래 연결된다.
3. 메타코딩(유튜브)은 트래픽 유입 채널로, 오픈스킬 브랜드로의 전환 게이트 역할을 한다.
4. 하나의 브랜드에 집중하면 마케팅 비용이 분산되지 않고, 한 서비스의 신뢰가 다른 서비스로 전이된다.

![오픈스킬 브랜드 아키텍처](images/brand-architecture.png)

### 각 서비스의 역할

| 서비스 | 핵심 역할 | 수익 모델 | 타깃 | 채널 |
|--------|----------|----------|------|------|
| 메타코딩 (유튜브) | 인지도, 트래픽 생성 | 광고 수익 | 일반/입문 | B2C |
| books.openskill.kr | 전문성 증명 매개체 | 특강/강의 (도서 판매는 부수적) | 독자/학습자 | B2C |
| edu.openskill.kr | AI 교육 에이전트 플랫폼 | 수강료/구독 | 학습자/개발자 | B2C |
| ncs.openskill.kr | 국가교육 기관 전용 프로그램 | 기관 계약 | 교육기관/정부기관 | B2G |
| author.openskill.kr | AI 집필 도구 | 월 구독 SaaS | 작가/출판사 | B2C |

### 메타코딩과 오픈스킬의 관계

메타코딩은 오픈스킬의 공식 서비스가 아니라 **트래픽 유입 채널** 이다. 별도의 독자적 정체성을 유지하면서 오픈스킬 서비스로 전환을 유도한다.

| 구분 | 메타코딩 | 오픈스킬 |
|------|---------|---------|
| 성격 | 개인 채널/커뮤니티 | 사업 브랜드 |
| 콘텐츠 | 무료 영상 | 유료 서적/교육/도구 |
| 연결 방식 | "더 깊이 배우려면 오픈스킬에서" | - |
| 브랜딩 | 독립 운영 | 서브도메인 통일 |

# 서비스 구조와 전환 경로

## 허브앤스포크 모델

오픈스킬의 4개 서비스는 **선형 퍼널로 이어지지 않는다.** 타깃, 채널, 진입 경로가 각각 다르기 때문이다.

- **books** = B2C 도서 출판. 수익 목적이 아닌 **저자 특강/강의로 연결하는 매개체**
- **edu** = B2C AI 교육 에이전트 플랫폼. 학습자가 직접 찾아오는 구조
- **ncs** = B2G 기관 전용 국가교육 프로그램. 영업/입찰 기반
- **author** = B2C AI 집필 도구. 작가/출판사 대상 SaaS. 독립 시장

따라서 "유튜브 -> books -> edu -> ncs -> author"처럼 일렬로 이어지는 퍼널이 아니라, **메타코딩이 허브가 되고 각 서비스가 독립적으로 뻗어나가는 구조** 다.

```
                      메타코딩 (유튜브)
                      인지 + 트래픽 허브
                            |
          +-----------------+-----------------+
          |                                   |
    books.openskill.kr                  edu.openskill.kr
    도서 출판 -> 특강/강의 연결          AI 교육 에이전트 (B2C)
    (책 = 매개체)

    author.openskill.kr               ncs.openskill.kr
    AI 집필 도구 (B2C)            기관 전용 프로그램 (B2G)
    -- 독립 SaaS 시장 --           -- 별도 영업 채널 --
```

## 서비스별 상세

### 메타코딩 (유튜브) -- 트래픽 허브

| 항목 | 내용 |
|------|------|
| 채널 | 유튜브 "메타코딩" |
| 콘텐츠 | 무료 강의, 튜토리얼, 기술 리뷰 |
| 핵심 지표 | 구독자 수, 조회수, 댓글 참여율 |
| 수익 | 광고 수익 (부수적) |
| 역할 | books와 edu로의 인지 확보 및 트래픽 전달 |

메타코딩은 오픈스킬의 인지도 엔진이다. 영상 설명란, 고정 댓글, 채널 탭에 books.openskill.kr과 edu.openskill.kr 링크를 배치한다. "이 주제를 체계적으로 정리한 책이 있습니다" / "AI 에이전트로 직접 실습해보세요"라는 자연스러운 연결을 만든다.

### books.openskill.kr -- 도서 출판 (매개체)

| 항목 | 내용 |
|------|------|
| 콘텐츠 | IT/AI 기술서적 (실물 + 전자책) |
| 핵심 지표 | 판매 부수, 특강 전환율, 리뷰 수 |
| 수익 | 도서 판매 자체가 아닌 **저자 특강/강의** 가 핵심 수익원 |
| 유입 경로 | 메타코딩 유튜브, 서점 검색, SNS |

**책은 돈을 벌기 위한 것이 아니다.** 책은 저자의 전문성을 증명하는 매개체이며, 이를 기반으로 저자 특강, 강의, 컨설팅으로 연결하는 것이 실제 비즈니스 모델이다.

```
메타코딩 영상 -> 도서 출판 -> 저자 특강/강의/컨설팅
(인지)          (신뢰 구축)    (핵심 수익)
```

도서 시리즈가 누적될수록 저자의 전문가 브랜딩이 강화되고, 특강/강의 섭외 단가와 빈도가 올라간다.

### edu.openskill.kr -- AI 교육 에이전트 플랫폼

| 항목 | 내용 |
|------|------|
| 콘텐츠 | AI 에이전트 기반 교육 플랫폼 |
| 핵심 지표 | 사용자 수, 학습 완료율, 재방문율 |
| 수익 | 수강료/구독 |
| 유입 경로 | 메타코딩 유튜브, 직접 검색, books 독자 |

교육하는 AI 에이전트를 만든 플랫폼이다. books의 독자가 "직접 해보고 싶다"는 욕구로 유입될 수 있지만, 이는 보조 경로이고 주된 유입은 직접 검색과 메타코딩을 통해 이루어진다.

### ncs.openskill.kr -- 기관 전용 국가교육 프로그램

| 항목 | 내용 |
|------|------|
| 콘텐츠 | 국가직무능력표준(NCS) 기반 교육/평가 |
| 핵심 지표 | 계약 기관 수, 수료자 수 |
| 수익 | 기관 계약 (B2G) |
| 유입 경로 | 입찰/영업, 기관 추천, 정부 사업 |

edu와는 완전히 다른 서비스다. B2G(정부/기관 대상) 채널이므로 메타코딩이나 books에서의 전환이 아니라 **별도의 영업 채널** 을 통해 운영된다. 오픈스킬 브랜드의 신뢰도가 간접적으로 도움을 줄 수 있다.

### author.openskill.kr -- AI 집필 도구

| 항목 | 내용 |
|------|------|
| 콘텐츠 | AI 집필 워크플로우 자동화 도구 |
| 핵심 지표 | MAU, 구독 전환율, 이탈률 |
| 수익 | 월 구독료 (SaaS) |
| 유입 경로 | Product Hunt, 작가 커뮤니티, 직접 마케팅 |

작가와 출판사를 대상으로 하는 독립적인 SaaS 제품이다. 오픈스킬 브랜드 아래 있지만, books/edu/ncs와 퍼널로 연결되지 않는다. 집필에이전트 시스템의 기술을 제품화한 것이므로 books와 **기술적 연관** 은 있으나, 사용자 전환 경로는 별개다.

## 전환 경로 정리

서비스 간의 연결은 선형 퍼널이 아니라 **부분적 크로스셀** 이다.

| From | To | 연결 방식 | 강도 |
|------|----|----------|------|
| 메타코딩 | books | 영상 설명란/고정댓글 링크 | 강함 (핵심 경로) |
| 메타코딩 | edu | 영상 내 실습 안내 | 중간 |
| books | 특강/강의 | 저자 전문성 기반 섭외 | 강함 (핵심 수익) |
| books | 메타코딩 | QR 코드로 보충 영상 연결 | 중간 |
| books | edu | 책 말미에 플랫폼 소개 | 약함 (보조 경로) |
| ncs | - | 별도 영업 채널 (B2G) | 독립 |
| author | - | 독립 SaaS 마케팅 | 독립 |

### 핵심 전환 경로 (실제 퍼널)

오픈스킬의 핵심 퍼널은 **메타코딩 -> books -> 저자 특강/강의** 다. 도서 판매 수익 자체가 목표가 아니라, 책이 저자의 전문성을 증명하고 특강/강의/컨설팅으로 연결하는 매개체 역할을 한다.

```
메타코딩 (인지) -> 도서 출판 (신뢰) -> 저자 특강/강의 (수익)
```

| 전환 구간 | 업계 평균 | 목표 |
|-----------|----------|------|
| 유튜브 시청 -> 구독 | 2~5% | 3% |
| 구독자 -> 도서 구매 | 1~3% | 2% |
| 도서 출판 -> 특강 섭외 | - | 도서 시리즈 누적에 비례 |

### 왜 책으로 돈을 벌지 않는가

| 비교 | 도서 판매 | 저자 특강/강의 |
|------|----------|-------------|
| 단가 | 권당 5,000~14,500원 | 회당 수십~수백만원 |
| 스케일 | 부수에 비례 (선형) | 브랜드에 비례 (지수적) |
| 마진 | 낮음 (인쇄/유통 비용) | 높음 (저자 인건비만) |
| 지속성 | 신간에 의존 | 카탈로그 누적으로 강화 |

도서 10권의 판매 수익보다 도서 10권의 저자가 받는 특강 1회 단가가 더 높을 수 있다. 책은 **비용이 아니라 투자** 다.

# 시리즈 네이밍 분석

## 한국 IT 출판사 시리즈 벤치마크

한국 주요 IT 출판사의 시리즈 네이밍 패턴을 분석한다.

### 행위/동사형

독자가 취할 행동을 시리즈명에 담는다. 가장 직관적이고 대중적인 패턴이다.

| 출판사 | 시리즈명 | 의미 |
|--------|---------|------|
| 길벗 | **Do it!** | 해봐! (행동 촉구) |
| 길벗 | **되는! 시리즈** | "이거면 된다" |
| 한빛미디어 | **혼자 공부하는** | 독학 강조 |
| 제이펍 | **Must Have** | 꼭 가져야 할 |
| 이지스퍼블리싱 | **모두의** | 누구나 |

### 은유/메타포형

추상적 이미지로 브랜드 정체성을 구축한다.

| 출판사 | 시리즈명 | 의미 |
|--------|---------|------|
| 한빛미디어 | **리얼타임** | 현장의 생생한 |
| 인사이트 | **프로그래밍 인사이트** | 통찰 |
| 에이콘 | **acorn+PACKT** | 글로벌 기술서 큐레이션 |
| 위키북스 | **위키북스 시리즈** | 출판사명 = 시리즈명 |

### 수준/깊이형

독자의 수준을 명시하여 선택을 돕는다.

| 출판사 | 시리즈명 | 의미 |
|--------|---------|------|
| 한빛미디어 | **Head First** | 뇌에 꽂히는 |
| 인사이트 | **만들면서 배우는** | 프로젝트 기반 |
| 프리렉 | **이것이 ~ 이다** | 정석/바이블 |

## 시리즈 네이밍 방향성

대화에서 논의된 핵심 키워드를 정리한다.

### 원하는 방향

- "AI 시대에 IT를 왜 배워야 하는지"를 담고 싶다
- AI의 발전 속도를 보는 개발자/일반인의 시각을 반영
- "출발이 다르다 / 다른 책과는 다르다 / AI 시대에 이것만큼 제대로된 책은 없다" 느낌
- "코딩이 필요없는 / 설계하는 / 개발자로 살아남기 / 노코딩" 방향도 고려
- "딸깍" 같은 간결한 느낌, "AI를 곁들인 / AI가 첨가된 / AI 한스푼" 같은 위트

### 네이밍 대안 6개 계열

#### 계열 1. "출발이 다르다" 계열

AI 시대라 남들과 시작점이 다르다는 포지셔닝.

| 후보 | 의미 |
|------|------|
| **제로 출발** | 0에서 시작하지만 도착이 빠르다 |
| **퍼스트 무브** | 먼저 움직이는 자 |
| **스타트 다름** | 출발선이 다르다 |

#### 계열 2. "AI를 곁들인" 계열

AI를 요리의 재료에 비유. 위트 있고 기억에 남는다.

| 후보 | 의미 |
|------|------|
| **AI 한 스푼** | 요리에 한 스푼 추가하듯 |
| **딸깍 시리즈** | 클릭 한 번으로 |
| **AI 첨가** | 인공 첨가물 대신 AI 첨가 |

#### 계열 3. "살아남기" 계열

AI 시대의 위기감을 자극하는 생존형 네이밍.

| 후보 | 의미 |
|------|------|
| **코드 서바이벌** | 코딩 생존기 |
| **디벨로퍼 넥스트** | 다음 세대 개발자 |
| **프루프 시리즈** | 미래를 대비하다 (Future-proof) |

#### 계열 4. "설계하는" 계열

코딩이 아닌 설계/아키텍처에 초점.

| 후보 | 의미 |
|------|------|
| **아키텍트** | 설계자 |
| **블루프린트** | 청사진 |
| **빌더** | 만드는 사람 |

#### 계열 5. "노코딩/핵심만" 계열

불필요한 코딩을 줄이고 핵심만 집중.

| 후보 | 의미 |
|------|------|
| **제로코드** | 코드 없이 |
| **코어만** | 핵심만 |
| **에센셜** | 본질만 |

#### 계열 6. "오픈스킬" 연계 계열

마스터 브랜드와 연결되는 시리즈명.

| 후보 | 의미 |
|------|------|
| **스킬업** | 능력 향상 |
| **오픈랩** | 열린 실험실 |
| **스킬트리** | 게임의 스킬트리처럼 |

## 네이밍 평가 매트릭스

주요 후보를 5가지 기준으로 평가한다. (5점 만점)

| 후보 | 직관성 | 차별성 | 확장성 | 기억력 | 브랜드 연계 |
|------|--------|--------|--------|--------|-----------|
| AI 한 스푼 | 4 | 5 | 3 | 5 | 2 |
| 딸깍 | 3 | 5 | 4 | 5 | 2 |
| 블루프린트 | 4 | 3 | 4 | 4 | 3 |
| 스킬트리 | 3 | 4 | 4 | 4 | 5 |
| 퍼스트 무브 | 4 | 3 | 4 | 3 | 2 |
| 코드 서바이벌 | 4 | 4 | 3 | 4 | 2 |

> 점수는 참고용이다. 최종 네이밍은 팀 논의 후 결정.

# 시리즈 정의 프레임워크

## 시리즈란 무엇인가

출판에서 시리즈는 **동일한 브랜드 정체성 아래 묶인 도서 라인업** 이다.

| 요소 | 역할 |
|------|------|
| 시리즈명 | 브랜드 인지, 수집 욕구 |
| 통일 표지 | 서점 진열 시 시각적 덩어리 효과 |
| 넘버링 | 순서/체계 전달 |
| 태그라인 | 시리즈의 핵심 약속 |

## 시리즈 구성 방식

IT 서적 시리즈는 크게 3가지 방식으로 구성할 수 있다.

### 방식 1. 주제별 (Topic-based)

각 권이 하나의 기술 주제를 다룬다.

```
[시리즈명] #1 -- Spring Boot
[시리즈명] #2 -- React
[시리즈명] #3 -- Docker
```

장점은 확장이 쉽고 독립 구매가 가능하다는 점. 단점은 시리즈 간 연결 스토리가 약하다는 점.

### 방식 2. 단계별 (Level-based)

한 주제를 깊이에 따라 나눈다.

```
[시리즈명] 입문 -- AI 개발 첫걸음
[시리즈명] 실전 -- 프로젝트로 배우는 AI
[시리즈명] 심화 -- AI 아키텍처 설계
```

장점은 학습 경로가 명확하다는 점. 단점은 주제 확장이 제한적이라는 점.

### 방식 3. 복합형 (주제별 + 단계별)

시리즈 안에 주제별 라인을 두고, 각 라인에 단계를 부여한다.

```
[시리즈명] 웹 입문 -- Spring Boot 첫걸음
[시리즈명] 웹 실전 -- Spring Boot 프로젝트
[시리즈명] AI 입문 -- LLM 활용 첫걸음
[시리즈명] AI 실전 -- AI 에이전트 만들기
```

**추천 방식.** 오픈스킬 북스의 첫 시리즈는 **주제별(Topic-based)** 방식으로 시작하고, 특정 주제의 인기가 검증되면 단계별로 확장하는 것이 리스크가 낮다.

## 시리즈 표지 브랜딩

시리즈의 시각적 통일감은 서점 진열 시 핵심 경쟁력이다.

### 통일 요소

| 요소 | 규칙 |
|------|------|
| 레이아웃 | 제목 위치, 저자명 위치 고정 |
| 색상 | 시리즈 대표색 1개 + 권별 포인트색 |
| 로고 | 오픈스킬 로고 위치 고정 |
| 시리즈 뱃지 | "[시리즈명]" 뱃지 위치 고정 |

### 권별 차별 요소

| 요소 | 변화 |
|------|------|
| 포인트 색상 | 권마다 다른 강조색 |
| 일러스트/아이콘 | 주제를 상징하는 이미지 |
| 부제 | 권별 고유 설명 |

### 참고: 한국 IT 출판사 표지 패턴

| 출판사 | 시각적 특징 |
|--------|-----------|
| 한빛미디어 | 오라일리 동물 세밀화 (번역), 밝고 컬러풀 (자체) |
| 제이펍 | 단색/2색 톤 + 기하학적 패턴 |
| 위키북스 | 단색 배경 + 큰 타이포그래피 |
| 길벗 | 밝은 색 + 캐릭터 ("Do it!" 시리즈) |
