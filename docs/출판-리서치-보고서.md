# Part A. 실물책 인쇄 규격

## A1. 판형 — B5 (182 x 257mm)

### "B5"란?

B5는 **국제 표준 종이 규격 (ISO 216)** 에서 정한 용지 크기다. 종이 규격에는 **A 시리즈** 와 **B 시리즈** 두 가지가 있다.

| 시리즈 | 원리 | 대표 예시 |
|--------|------|----------|
| **A 시리즈** | A0(면적 1m²)를 반으로 접어가며 번호 부여 | A4(210x297), A5(148x210) |
| **B 시리즈** | A 시리즈 사이 크기를 채우기 위해 만든 규격 | **B5 (182x257)** , B4 (257x364) |

B5는 **A4보다 한 단계 작고, A5보다 한 단계 큰 크기**다. 대학교 전공 교재가 대체로 B5에 가깝다.

![판형 크기 비교](images/size-comparison.png)

---

현재 PDF 빌드 설정 (Typst) 은 46배판 (188x257mm) 으로 B5에 가깝다.

| 판형 | 크기 (mm) | 용도 | 비고 |
|------|----------|------|------|
| **B5** | **182 x 257** | IT 기술서적, 전공 교재 | 현재 설정과 거의 동일 |
| 46배판 | 188 x 257 | B5와 유사하나 가로 6mm 넓음 | 현재 PDF 빌드 설정 |
| 신국판 | 152 x 225 | IT 서적 표준 (한빛, 제이펍) | 대안 |

> 🔍 **확인 필요**: 부크크/교보 바로출판에서 B5(182x257mm)를 정확히 지원하는지, 아니면 가장 가까운 규격으로 맞춰야 하는지 각 플랫폼에서 직접 확인

> 📞 **업체 확인**: 인쇄소(북토리 등)에 B5 정확한 치수 전달 시 별도 비용이 드는지

### B5 vs 신국판 비교

| 항목 | B5 (182x257) | 신국판 (152x225) |
|------|-------------|-----------------|
| 가로 | 182mm | 152mm (30mm 좁음) |
| 세로 | 257mm | 225mm (32mm 짧음) |
| 코드 블록 가로폭 | 넉넉함 | 좁음 (줄바꿈 더 자주 발생) |
| 다이어그램 공간 | 넉넉함 | 제한적 |
| 인쇄 단가 | 조금 더 비쌈 (종이 더 큼) | 표준이라 저렴 |
| 휴대성 | 좀 큼 | 적당함 |

**IT 기술서적에 B5가 좋은 이유** : 코드 블록이 많은 책은 가로폭이 넓어야 줄바꿈이 덜 발생한다.

### TIP: A4 용지와 비교하면?

우리가 매일 쓰는 A4 용지와 비교하면 크기감이 잡힌다.

| 판형 | 가로 | 세로 | A4 대비 |
|------|------|------|---------|
| **A4 (참고)** | **210mm** | **297mm** | 기준 |
| B5 | 182mm | 257mm | A4보다 가로 28mm 좁고, 세로 40mm 짧음 |
| 신국판 | 152mm | 225mm | A4보다 가로 58mm 좁고, 세로 72mm 짧음 |

- B5는 A4의 약 **87% 가로, 86% 세로** → A4를 조금 줄인 느낌
- 신국판은 A4의 약 **72% 가로, 76% 세로** → 일반 소설책 크기에 가까움
- 한빛미디어/제이펍 기술서적을 손에 들어보면 = 신국판 느낌
- 대학교 전공 교재를 떠올리면 = B5에 가까운 느낌

## A2. 인쇄소 입고 포맷 — PDF

**PDF가 유일한 표준.** InDesign, PostScript 등은 불필요.

### 인쇄 방식별 어디를 통하는지

작가가 선택하는 것은 **"어디를 통해 인쇄할 것인가"** 다. 크게 2가지 길이 있다.

**길 1. 셀프출판 플랫폼 (POD)** — 부크크, 교보 퍼플(바로출판), 유페이퍼 등
- 웹사이트에 PDF를 업로드하면 끝. 주문이 들어올 때마다 1부씩 디지털 프린터로 찍어줌
- 요구 PDF: **일반 PDF** (폰트 포함만 하면 됨). 플랫폼이 내부적으로 변환해줌
- 장점: 초기 비용 0원, 재고 없음, 가장 쉬움
- 단점: 권당 단가 높음 (5,000~8,000원)
- ✅ **대표 업체**: [부크크](https://bookk.co.kr/), [교보 퍼플(바로출판)](https://product.kyobobook.co.kr/pod/main), [유페이퍼](https://www.upaper.net/)

**길 2. 인쇄소 직접 의뢰 (오프셋)** — 북토리, 나눔인쇄, 열림프린팅 등
- 인쇄소에 직접 연락 → 견적 → PDF 파일 전달 → 200~500부 이상 한 번에 찍음
- 요구 PDF: **PDF/X-1a** (CMYK 전용, 폰트 완전 포함, 투명 효과 없음)
- 왜 엄격한가: 판(plate)을 떠서 대량으로 찍으므로 실패 = 종이+잉크+시간 전부 날림
- 장점: 수량 많으면 단가 절반 이하
- 단점: 초기 비용 200만원+, 재고 보관
- 📞 **견적 필요**: [북토리](https://booktory.com/), 나눔인쇄, 열림프린팅

**초기 방향**: POD → 부크크 등 활용

### PDF 규격 용어 설명

| 용어 | 뜻 |
|------|-----|
| **PDF/X** | 인쇄 전용 PDF 국제 표준. "인쇄소 기계가 100% 해석 가능"을 보장하는 엄격한 규격 |
| **PDF/X-1a** | 가장 엄격. 색상은 반드시 CMYK 잉크만, 투명 효과 금지, 폰트 전부 포함. 오프셋 인쇄소가 요구 |
| **PDF/X-4** | 유연한 규격. 투명 OK, RGB 일부 OK. 디지털/POD용 |
| **CMYK** | 인쇄기의 잉크 4색. **C**yan(하늘색) + **M**agenta(분홍색) + **Y**ellow(노란색) + **K**ey(검정). 이 4가지 잉크를 섞어 모든 색을 만든다. 모니터(빛 RGB)와 인쇄기(잉크 CMYK)는 원리가 달라서 화면 색과 인쇄 색이 미묘하게 다를 수 있다. 형광색/네온색은 인쇄하면 탁해진다. 본문 흑백이면 신경 안 써도 됨 |
| **폰트 임베딩** | PDF 안에 사용한 글꼴을 함께 넣는 것. 안 넣으면 인쇄소 PC에 폰트 없을 때 글자 깨짐 |
| **300dpi** | 1인치에 점 300개. 화면은 72~96dpi면 충분하지만, 인쇄는 300dpi 이상이어야 선명 |
| **Bleed (재단 여백)** | 인쇄 후 칼로 자를 때 오차 대비 여유분 3mm. 잘려도 흰 테두리가 안 보이게 |
| **Gutter (제본 여유)** | 책 묶는 쪽(안쪽) 여백. 제본하면 안쪽이 접혀서 글자가 가려지므로 넉넉하게 |
| **오프셋 인쇄** | 판(plate)을 만들어 찍는 대량 인쇄. 초기 비용 높지만 수량 많으면 단가 낮음 |
| **POD** | Print On Demand(주문형). 주문마다 1부씩 디지털 프린터로 인쇄. 재고 없음, 단가 높음 |

## A3. 인쇄 사양

| 항목 | 값 | 비고 |
|------|-----|-----------|
| 판형 | **B5 (182x257mm)** | IT 기술서적 가로폭 유리 |
| 제본 | **무선제본** (풀 제본) | 책등에 풀을 발라 붙이는 방식 |
| 본문 용지 | 미색모조 100g | 🔍 인쇄소별 확인 |
| 표지 용지 | 스노우지 250g + 무광 코팅 | 🔍 부크크 옵션 확인 |
| 본문 도수 | **1도 (흑백)** | 코드 중심이므로 흑백 |
| 표지 도수 | **4도 (풀컬러)** | |
| 페이지 수 | **16의 배수** 권장 | 1대수 = 16페이지 |

### "미색모조지"란?

**미색모조지** = 약간 누런빛 (아이보리색) 이 도는, 코팅 없는 일반 종이. 눈이 덜 피로하고 가독성이 좋아서 **글자가 많은 책의 표준 용지** 다.

| 종이 | 색감 | 표면 | 용도 |
|------|------|------|------|
| **미색모조지** | 아이보리 (약간 노란빛) | 무광, 코팅 없음 | 소설, 기술서적, 교과서 본문 |
| 백색모조지 | 순백색 | 무광, 코팅 없음 | 사무용, 복사 용지 |
| 아트지 | 백색 | 유광, 코팅 있음 | 잡지, 사진집, 카탈로그 |

**100g** = 평량 (1m² 크기 종이 한 장의 무게). 기술서적 본문에는 100g이 표준이다.

| 평량 | 느낌 | 용도 |
|------|------|------|
| 70g | 얇음, 뒷면이 약간 비침 | 문고판, 페이지 많은 책 |
| 80g | 일반 복사 용지 수준 | 사무용 |
| **100g** | 적당한 두께, 안 비침 | **기술서적 본문 표준** |
| 120g | 두꺼움 | 고급 서적, 화보집 |

### "도수"란?

**"도"는 인쇄에 사용하는 잉크 색상 수.** "몇 가지 색 잉크로 찍을 건가?"를 뜻한다.

| 표기 | 뜻 | 설명 | 비용 |
|------|-----|------|------|
| **1도** | 잉크 **1가지** (검정) | 흑백 인쇄. 글자+코드만 있는 본문에 적합 | 저렴 |
| **2도** | 잉크 **2가지** (검정 + 특색 1개) | 강조색 하나 추가 (예: 파란색 제목) | 중간 |
| **4도** | 잉크 **4가지** (C+M+Y+K) | 풀컬러 인쇄. 사진/컬러 다이어그램 표현 | 비쌈 |

- 본문 1도(흑백) + 표지 4도(풀컬러) 조합이 기술서적의 표준
- 본문도 4도로 하면 인쇄비가 2~3배 뛰므로, 본문에 컬러가 꼭 필요한 경우에만 선택

### "16의 배수"란?

맞다. 16p, 32p, 48p, ..., 160p, 176p, 192p 같은 숫자를 말한다.

**이유**: 인쇄는 **큰 종이 한 장에 여러 페이지를 한꺼번에** 찍는다.

![오프셋 인쇄 구조](images/printing-layout.png)

- 1대수 = 큰 종이 2장 양면 인쇄 후 접기 = **16페이지**
- 총 160페이지 책 = 10대수 = 딱 맞음
- 총 170페이지 책 = 10.625대수 = 마지막 대수에 **6페이지가 빈 페이지**로 남음

빈 페이지가 생기면 종이 낭비 + 책이 어색해 보일 수 있다. 그래서 **부록이나 메모 페이지를 넣어서 16의 배수로 맞추는 것**이 관행이다.

> 🔍 **확인 필요**: 부크크 같은 POD는 1부씩 디지털 프린터로 찍으므로 16배수 제약이 없을 수 있다. POD에서는 신경 안 써도 되는지 확인 필요

## A4. 여백/마진

### "Gutter (거터)" 란?

**Gutter** = 책의 **제본 쪽 (안쪽) 여백** . 책을 펼치면 가운데가 풀로 묶여 있어서 완전히 펴지지 않는다. 이 접히는 영역에 글자가 들어가면 가려져서 안 보인다. 그래서 제본 쪽 여백을 바깥쪽보다 넓게 잡는데, 이 넓어진 안쪽 여백을 Gutter라고 부른다.

| 용어 | 위치 | 설명 |
|------|------|------|
| **바깥쪽 여백 (Outer)** | 책 가장자리 쪽 | 손으로 잡는 쪽 여백 |
| **Gutter (Inner)** | 책 중앙 제본 쪽 | 풀로 묶인 쪽 여백. 바깥쪽보다 넓어야 함 |

> **핵심** : 무선제본은 안쪽이 말려 들어가므로, Gutter를 바깥쪽보다 **최소 5~10mm 더 넓게** 줘야 한다. 안 그러면 독자가 책을 억지로 벌려야 코드를 읽을 수 있다.

### 마진 설정 (B5 기준)

먼저 업계 표준 **3가지 근거**를 정리하고, 우리 책에 적용할 값을 도출한다.

---

### 근거 1: Langham Press 인쇄소 공식 기술 가이드

> ✅ 출처: [Langham Press - Perfect Bound Gutter Margins (PDF)](https://www.langhampress.co.uk/assets/downloads/Perfect_Bound_Setup_2019.pdf)

영국 인쇄소 Langham Press의 기술 문서에서 **가장 명확한 수치**를 제시한다.

**원문**: "the spine gutter margins should be at least **7mm wider** than the foredge margins"
(안쪽 여백은 바깥쪽 여백보다 **최소 7mm 더 넓어야** 한다)

**예시**: 바깥쪽(상하좌우) 15mm → 안쪽(spine) 15mm + 7mm = **22mm**

| 위치 | 왼쪽 페이지 (짝수) | 오른쪽 페이지 (홀수) |
|------|-------------------|-------------------|
| 바깥쪽 여백 | 15mm (왼쪽) | 15mm (오른쪽) |
| 본문 텍스트 영역 | 중앙 | 중앙 |
| 안쪽 여백 (제본 쪽) | 22mm (오른쪽) | 22mm (왼쪽) |
| **안쪽 - 바깥쪽 차이** | **+7mm** | **+7mm** |

**이유**: 무선제본은 풀로 묶기 때문에 안쪽이 "nipped together"(눌려 붙음) 된다. 7mm를 추가하지 않으면 텍스트가 제본 쪽으로 빨려 들어가 보이고, 시각적으로 중앙 정렬이 안 됨.

---

### 근거 2: Amazon KDP 공식 규격 (페이지 수별)

> ✅ 출처: [Amazon KDP - Trim Size, Bleed, and Margins](https://kdp.amazon.com/en_US/help/topic/GVBQ3CMEQW3W2VL6)

| 페이지 수 | 안쪽 여백 최소 | 바깥쪽 여백 최소 | **안쪽이 바깥보다 넓은 정도** |
|-----------|-------------|----------------|---------------------------|
| 24~150p | 9.6mm | 6.4mm | **+3.2mm** |
| 151~300p | 12.7mm | 6.4mm | **+6.3mm** |
| 301~500p | 15.9mm | 6.4mm | **+9.5mm** |
| 501~700p | 19.1mm | 6.4mm | **+12.7mm** |
| 701~828p | 22.3mm | 6.4mm | **+15.9mm** |

페이지가 많을수록 책이 두꺼워져 안쪽이 더 안 펴짐 → gutter 차이가 커짐.
KDP는 **최소값**이다. 실제로는 더 넓게 잡는 것이 일반적.

### 근거 3: Printivity 인쇄 가이드

> ✅ 출처: [Printivity - How to Design for Gutter Margins](https://www.printivity.com/insights/how-to-design-for-gutter-margins)

**원문**: "0.5 inches (12.7mm) for your gutter margin is the best recommendation for most books"
"Perfect bound books do not lay flat when opened and tend to curve towards the spine"

안쪽에 **추가로 1/4"~1/2" (6.4~12.7mm)** 의 safety margin을 더 줘야 한다고 권장.

### 종합: 우리 책에 적용할 값 (B5, 약 200p)

| 항목 | 수치 | 근거 |
|------|------|------|
| 재단 여백 (Bleed) | 상하좌우 **3mm** | 인쇄 표준 |
| 안전 영역 | 재단선 안쪽 **5mm** | 텍스트/이미지 잘림 방지 |
| **안쪽 여백 (Gutter)** | **22mm** | Langham Press 예시 (15+7=22). KDP 최소 12.7mm보다 여유 |
| 바깥쪽 여백 | **15mm** | Langham Press 예시 기준 |
| 위 여백 | **15mm** | |
| 아래 여백 | **18mm** | 페이지 번호 공간 |
| **안쪽 - 바깥쪽 차이** | **7mm** | Langham Press "최소 7mm" 권장 |

> **검증 결과** : "안쪽을 바깥쪽보다 5~10mm 더 넓게"는 업계 표준과 일치한다.
> - Langham Press: 최소 **7mm** 더 넓게 → 5~10mm 범위의 정중앙
> - KDP 200p: **6.3mm** 더 넓게
> - KDP 300p: **9.5mm** 더 넓게
>
> **결론** : "추가로 5~10mm를 더한다"가 아니라, **최종 결과에서 안쪽이 바깥보다 약 7mm 넓은 상태** 가 정답이다.

> 📞 **확인 필요**: 부크크에서 B5 무선제본 시 권장하는 정확한 gutter 값을 직접 확인. 위 수치는 해외 인쇄소 기준이므로 한국 인쇄소는 다를 수 있음

### 왜 안쪽 여백이 더 넓어야 하는가? (양쪽 펼침 원리)

책을 펼치면 **왼쪽 페이지 (짝수) 와 오른쪽 페이지 (홀수)** 가 가운데에서 만난다. 이 가운데 부분이 제본으로 묶여 있어서 물리적으로 완전히 펴지지 않는다.

| 페이지 | 바깥쪽 여백 | 본문 | 안쪽 여백 (제본 쪽) |
|--------|-----------|------|-------------------|
| 왼쪽 (짝수, 42p) | 15mm (왼쪽) | 텍스트 영역 | 22mm (오른쪽) |
| 오른쪽 (홀수, 43p) | 15mm (오른쪽) | 텍스트 영역 | 22mm (왼쪽) |
| | | **제본선** | |

제본선 근처 약 5~10mm는 풀로 묶여 굽어서 안 보인다.

**원리**: 무선제본(풀 제본)은 책등에 풀을 발라 종이를 묶는다. 책을 아무리 힘껏 펼쳐도 가운데 제본선 근처 약 5~10mm는 굽어서 안 보인다. 그래서:

- **오른쪽 페이지 (홀수)** : 왼쪽 (안쪽) 이 제본선에 가까움 → **왼쪽 여백을 넓게** (22~25mm)
- **왼쪽 페이지 (짝수)** : 오른쪽 (안쪽) 이 제본선에 가까움 → **오른쪽 여백을 넓게** (22~25mm)
- 바깥쪽 여백은 상대적으로 좁아도 됨 (15~20mm)

이것이 **"비대칭 마진"** 또는 **"미러 마진"** 이라고 부르는 설정이다. Typst에서는 `inside` / `outside` 마진으로 설정 가능.

모든 인쇄 가이드에서 동일하게 권장하는 원리다.

> ✅ 출처: [Kindlepreneur - Book Gutter](https://kindlepreneur.com/book-gutter/), [Book Designer - Page Margins](https://www.thebookdesigner.com/book-layouts-page-margins/)

**만약 안쪽 여백을 바깥쪽과 같게 하면?** 독자가 코드 블록을 읽으려고 책을 억지로 벌려야 한다. 특히 코드가 있는 기술서적에서는 치명적이다.

### 페이지 수와 Gutter의 관계

페이지가 많을수록 제본 부분이 두꺼워져 더 안 펴진다. 그래서 페이지 수에 따라 Gutter를 다르게 설정한다.

| 페이지 수 | 추천 안쪽 여백 (Gutter) | 이유 |

| ~150p | 18~20mm | 얇아서 비교적 잘 펴짐 |
| 150~300p | 20~22mm | 표준 두께 |
| 300p+ | 22~25mm | 두꺼워서 가운데가 많이 가려짐 |

## A5. 코드 블록 가독성

> **핵심** : 책 페이지는 모니터보다 가로폭이 좁다. 긴 코드가 강제 줄바꿈되지 않도록 코드를 적절히 끊고, 반드시 **고정폭 폰트 (Monospace)** 를 적용해야 한다.

### 현재 시스템 대응 상태

| 항목 | 현재 PDF 빌드 설정 | 요구 사항 | 상태 |
|------|----------------|-----------|------|
| 코드 폰트 | Menlo 8pt | 고정폭 폰트 | ✅ OK |
| 코드 줄 길이 | 제한 없음 | 책 폭에 맞춰 끊기 | ⚠️ 수동 또는 자동 줄바꿈 필요 |
| 줄바꿈 처리 | breakable: true | 자동 페이지 넘김 | ✅ OK |

### B5 기준 코드 줄 길이 계산

```
B5 가로 = 182mm
- 안쪽 여백 22mm - 바깥쪽 여백 18mm = 코드 영역 142mm
- Menlo 8pt 기준 글자 폭 ≈ 1.8mm
- 최대 글자 수 = 142 / 1.8 ≈ 78자
```

🔍 **확인 필요**: 실제 Typst 출력에서 코드 블록 최대 글자 수를 PDF로 빌드해서 확인해야 함

### 대응 전략

1. versions/ 코드 원본에서 **줄 길이 70자 이내**로 유지 (여유 포함)
2. Typst에서 코드 블록 `pre-wrap` 설정으로 자동 줄바꿈 보험
3. D2Coding 폰트도 대안으로 검토 (한글 주석 지원 우수)

## B2. 책등 너비 계산

### 공식

```
책등 두께(mm) = (총 페이지 수 / 2) x 종이 한 장 두께(mm) + 0.5mm(풀칠)
```

| 평량 | 한 장 두께 |
|------|-----------|
| 70g | 0.087mm |
| 80g | 0.093mm |
| **100g** | **0.113mm** |
| 120g | 0.134mm |

> ✅ 출처: [프린트올데이 - 종이 상식](https://printallday.com/properties-of-paper/)

# Part C. 전자책

## C1. 전자책 형태

> **방향** : 전자책도 **PDF 형태** 로 판매한다. EPUB은 현재 만들지 않는다.

IT 기술서적은 코드 블록과 다이어그램이 많아서 PDF 전자책이 유리하다. 레이아웃이 인쇄본과 동일하게 유지되기 때문. 위키북스, 한빛미디어도 IT 서적 전자책을 PDF로 판매한다.

| 출판사 | 전자책 형태 | 비고 |
|--------|-----------|------|
| 위키북스 | **PDF 포맷** | 코드 레이아웃 유지 |
| 한빛미디어 | **PDF** (DRM-free, 워터마크) | 리얼타임 시리즈 |

## C2. 실물책 PDF vs 전자책 PDF 차이

같은 PDF라도 인쇄용과 전자책용은 약간 다르다.

| 항목 | 인쇄용 PDF | 전자책 PDF |
|------|----------|-----------|
| Gutter (안쪽 여백) | 22mm (제본 대비) | 좌우 대칭 (제본 없음) |
| 이미지 해상도 | 300dpi | 150dpi면 충분 (파일 크기 절감) |
| 재단 여백 (Bleed) | 3mm 필요 | 불필요 |
| 파일 크기 | 클 수 있음 | 가능하면 작게 |

> 🔍 **확인 필요** : 부크크/교보에서 전자책을 어떤 포맷으로 등록하는지

## C3. 향후 EPUB 전환 시 참고사항

현재는 PDF만 만들지만, 나중에 EPUB도 만들 경우를 위한 메모.

### EPUB이란?

- **EPUB** = 웹페이지처럼 화면 크기에 맞춰 글이 자동으로 흘러가는 (리플로우) 전자책 형식
- 독자가 글자 크기를 키우면 한 줄에 들어가는 글자 수가 줄고 페이지 수가 늘어남
- 스마트폰, 태블릿 등 다양한 화면에서 읽기 편함
- 단점: 코드 블록이 깨질 수 있음

### EPUB 전환 시 필요한 작업

- [ ] 하나의 Markdown 원본에서 `pandoc → EPUB3` 빌드 파이프라인 구축
- [ ] EPUB에 모노스페이스 폰트 임베딩 (NanumGothicCoding 등) — 코드 블록용 폰트를 EPUB 안에 함께 넣기
- [ ] 코드 줄 60~70자 이내 유지 (작은 화면 대비)
- [ ] CSS에서 `pre-wrap` 설정 (자동 줄바꿈)
- [ ] Markdown에서 "아래/위" 같은 위치 표현 대신 "다음" 사용 (EPUB에서는 위치가 바뀔 수 있음)

## C4. 전자책 표지 규격

### 플랫폼별 표지 사이즈

| 플랫폼 | 사이즈 (px) | 비율 | 파일 형식 | 비고 |
|--------|-----------|------|---------|------|
| **부크크** | 550x720 (EPUB) | 1:1.31 | PDF/EPUB | 부크크 마켓에서만 판매 |
| **교보 바로출판** | 가로 1080px 이상, 뷰어 700x1000 | 1:1.43 | JPG | 교보+YES24+알라딘+리디 자동 유통 |
| **유페이퍼** | 700x1000 | 1:1.43 | JPG/PNG | 교보/YES24/알라딘/리디 자동 유통 |
| **리디북스** | 최대 1280x1952 | 1:1.53 | JPG | 리디셀렉트/리디북스 |
| **아마존 KDP** | **1600x2560** | 1:1.6 | JPEG | 글로벌 표준 (참고용) |

> **실무 팁**: **1600x2560px**로 만들어두면 모든 플랫폼에서 다운사이징으로 대응 가능하다.

### 실물책 표지 vs 전자책 표지

| 항목 | 전자책 | 실물책 |
|------|------|------|
| 구성 | **앞표지만** | 앞표지 + 책등 + 뒷표지 스프레드 |
| 해상도 | 72~150 DPI | 300 DPI |
| 색상 모드 | **RGB** | **CMYK** |
| 블리드 (재단 여백) | 불필요 | 3mm 필수 |
| 책등 | 불필요 | 필수 (페이지 수에 따라 두께 변동) |
| 뒷표지 | 불필요 | 필수 (ISBN 바코드, 소개문) |
| 파일 형식 | JPEG/PNG | PDF |

### 플랫폼별 전자책 유통 범위 비교

부크크 전자책은 **부크크 자체 마켓에서만** 판매된다. 교보/YES24/알라딘 등 메이저 서점 유통이 필요하면 **퍼플** 또는 **유페이퍼**를 이용해야 한다.

| 플랫폼 | 전자책 유통 범위 | 인세 | 포맷 |
|--------|--------------|------|------|
| **부크크** | 부크크 마켓만 | 35% | PDF/EPUB |
| **퍼플 (교보)** | 교보+YES24+알라딘+리디+Google Play+iBooks 등 **17곳** | 50~60% | EPUB/PDF |
| **유페이퍼** | 교보+YES24+알라딘+리디 | **70%** | EPUB |

**퍼플의 전자책 유통 강점.** ISBN을 무료로 발급해주고, 국내 17곳 이상 + 해외(Google Play, iBooks)까지 자동 유통한다. 전자책 유통 범위가 가장 넓다.

**유페이퍼의 인세 강점.** 전자책 인세가 70%로 가장 높다. 다만 종이책 POD는 지원하지 않는다.

> **추천 전략**: 종이책은 **부크크** (자체몰 35% 수익), 전자책은 **퍼플** (유통 범위) 또는 **유페이퍼** (인세율)로 투트랙 운영.

> 📞 **업체 문의**: 교보 퍼플 POD 종이책 표지 스프레드 규격 상세

# Part D. 수익구조 + 유통

## D1. 수익구조 분석

### 정가 15,000원 기준 (전통 출판, 서점 유통)

> **1단계: 서점 수수료**
> - 대형 서점(교보, YES24)은 정가의 35~40% 마진
> - 1인 출판사 공급률: **60%**
> - 15,000원 × 60% = **9,000원** (출판사 매출)
> - 서점이 6,000원 가져감
>
> **2단계: 원가 빼기** (9,000원에서)
> - 작가 인세 10% = 1,500원
> - 인쇄비 = ~3,000원 (300p, 1,000부)
> - 물류비 = ~1,000원 (파주 창고+배송)
>
> **3단계: 최종**
> - 출판사 순수익 = 9,000 - 1,500 - 3,000 - 1,000 = **3,500원/권**
> - 작가 인세 = **1,500원/권**
> - 합산 (작가=출판사일 때) = **5,000원/권 (33%)**

> **핵심** : 서점을 통하지 않고 직판하면 수익이 극대화된다.

# Part F. 작가 AX (AI Transformation)

## F1. AX 비전

> **핵심** : 현재 집필에이전트 시스템을 제품화하면 온라인 출판사를 운영할 수 있다.

AX = AI Transformation (DX의 다음 단계). 현재 집필에이전트 시스템을 **작가와 AI가 대화하며 책을 만드는 제품** 으로 발전시키는 것.

## F2. 시장 분석 — 핵심 발견

**작가의 페르소나, 동기부여, 워크플로우를 종합하여 다방면으로 집필을 지원하는 AI 에이전트** 는 시장에 존재하지 않는다.

| 서비스 | 가격 | 한계 |
|--------|------|------|
| Sudowrite | $10~59/월 | 소설 전용, 워크플로우 없음 |
| Jasper | $39~59/월 | 마케팅 콘텐츠 전용 |
| 에피소드 | 45,000원/월 | 에세이/자기계발, IT 서적 부적합 |

기존 도구는 모두 "빈 페이지에 AI가 보조"하는 수준.
**7단계 워크플로우 + 멀티에이전트 + 작가 페르소나/동기부여를 종합한 다방면 집필 지원** = 시장에 없음 = 기회.

## F3. 현재 시스템 자산

| 자산 | 내용 |
|------|------|
| 7단계 워크플로우 | 씨앗 → 코드해부 → 시나리오 → 뼈대 → 집필 → 프롤로그 → 마무리 |
| 5개 에이전트 | analyst-architect, writer, editor, illustrator, publisher |
| 22개 스킬 | 코드분석, 기획, 집필, 검토, 시각화, 인쇄 |
| 인쇄 파이프라인 | Markdown → Typst → PDF (6단계 자동) |
| 실증 | 158페이지 기술서적 1권 완성 |

## F4. 기술 스택 (제품화 시)

| 레이어 | 추천 | 이유 |
|--------|------|------|
