# 씨앗

## 주제
RAG 기반 사내 AI 비서 만들기 — LLM + 벡터DB + FastAPI로 사내 문서 검색 AI 구축

## 집필 동기
기존 RAG 관련 자료는 대부분 이론 설명이나 조각난 예제에 머문다. "Retriever가 뭔지"는 알겠는데, 실제로 사내 문서를 넣고 질문하면 어떻게 되는지, 정확도가 안 나올 때 뭘 건드려야 하는지를 보여주는 자료는 드물다. 이 책은 LLM 환각을 직접 체험하는 것에서 시작해서, 문서를 파싱하고, 벡터화하고, RAG 엔진을 만들고, 에이전트로 통합하고, 성능을 튜닝하는 전 과정을 하나의 프로젝트로 연결한다.

## 저자 경험
> AI가 범용적으로 작성 — 특정 저자의 경험을 전제하지 않음.

이 기술을 처음 접하는 사람이 겪는 전형적 경험으로 서술한다. LLM에게 물어봤는데 엉뚱한 답이 나오고, 문서를 넣어봤는데 검색 정확도가 안 나오고, 청킹 사이즈를 바꿔봐도 뭐가 좋은 건지 모르겠는 — 그런 보편적인 시행착오를 이야기의 뼈대로 삼는다.

## 핵심 메시지
"직접 만들어봐야 이해한다" — 이론 100번 읽는 것보다 환각을 한 번 보고, 코드를 한 번 짜고, 튜닝을 한 번 돌려보는 게 낫다.

## 책의 의도
**이해 중심의 스토리텔링**으로 RAG 전체 파이프라인을 설명한다. 이 책은 코드를 따라치는 튜토리얼이 아니라, "왜 이게 필요한지"를 이야기로 풀어내는 개념서다. CH03~CH10 전체를 다루되, 각 기술이 등장하는 맥락과 이유를 스토리텔링으로 전달하는 것이 핵심. 에이전트 통합(CH08), 운영(CH09), 튜닝(CH10)도 이해 중심으로 포함한다.

## 의도 밖 범위
- **Docker/PostgreSQL 세팅 상세**: 설치/설정은 간략 안내만. docker-compose up 수준.
- **Langfuse/모니터링 상세**: 존재는 언급하되 연동 설정 상세는 제외.
- **CSS/Jinja2 프론트엔드 UI 구현**: 웹 UI 코드는 제외. API 중심으로 설명.

## 차별점
1. **하나의 프로젝트, 처음부터 끝까지**: 조각난 예제가 아니라, 사내 AI 비서라는 하나의 프로젝트를 CH03에서 CH10까지 점진적으로 성장시킨다.
2. **실패부터 시작**: 각 챕터가 "정답"부터 알려주지 않고, LLM 환각 → RAG 필요성 같은 실패 체험에서 출발한다.
3. **RAG 튜닝까지**: 대부분의 RAG 자료가 기본 구축에서 멈추는데, 이 책은 10가지 튜닝 실험과 평가 프레임워크까지 다룬다.
4. **이야기로 읽는 기술서**: 교과서가 아니라 스토리텔링. 비유와 시나리오로 개념을 풀어 설명한다.
