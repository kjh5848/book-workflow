## STEP 1: 씨앗

- **주제**: RAG 기반 사내 AI 비서 만들기 — LLM + 벡터DB + FastAPI로 사내 문서 검색 AI 구축
- **집필 동기**: 기존 RAG 자료가 너무 이론적이다 — 실전 프로젝트로 처음부터 끝까지 만들어보는 자료가 부족
- **저자 경험**: AI가 범용적으로 작성 — "처음 접하는 사람의 전형적 경험"으로 작성
- **핵심 메시지**: "직접 만들어봐야 이해한다" — 환각 체험 → 코드 작성 → 튜닝 실험까지 직접 해보는 여정
- **책의 의도**: RAG 전체 파이프라인 이해 — 문서 파싱 → 청킹 → 임베딩 → 검색 → 답변 생성의 흐름을 개념적으로 이해
- **의도 밖 범위**: Docker/PostgreSQL 세팅 상세, Langfuse/모니터링 상세, CSS/Jinja2 프론트엔드 UI 구현

### CH02 변수 결정
- **이번 버전**: v0.1 — FastAPI로 직원/연차/매출 CRUD 시스템
- **이전 버전에서 변경**: v0.0(더미 문서 RAG) → v0.1(실제 PostgreSQL DB 기반 사내 시스템)
- **왜 해야 하나**: RAG로 문서 검색은 되지만, "김대리 연차 몇 개?"처럼 실시간 DB 데이터를 물어보면 답을 못 한다. AI 비서가 조회할 사내 시스템을 먼저 만들어야 한다.
- **비유**: 식당 비유 — API는 식당 웨이터, DB는 주방, 요청(주문)을 받아 데이터(음식)를 가져다준다
- **핵심 개념**: REST API, Pydantic, psycopg2, CRUD 패턴
- **실패담**: 별도 없음 — 시나리오 흐름 속에서 자연스럽게
- **한 줄 기억**: "AI 비서가 조회할 사내 시스템을 만들었다. API는 웨이터처럼 요청을 받아 DB에서 데이터를 가져다준다."
