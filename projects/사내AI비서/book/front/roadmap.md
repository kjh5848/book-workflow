# 여정의 지도 — 무엇을 만들고, 무엇을 배우는가

---

이 책은 총 7개의 챕터로 이루어집니다. 챕터가 끝날 때마다 비서가 한 단계 성장합니다. 이 페이지는 그 여정 전체를 한눈에 보여주는 지도입니다.

---

## 전체 여정

```
시작 ──────────────────────────────────────────── 완성
  │
  │   Part 1: 재료 준비
  ├── Ch.1  회사 장부를 열어보자
  │         └─ 비서가 볼 수 있는 장부(DB)가 생겼다
  │
  ├── Ch.2  ChatGPT에게 사내 문서를 보여줬더니
  │         └─ LLM의 한계를 직접 확인하고, RAG 방향을 잡았다
  │
  ├── Ch.3  서류함을 뒤져보자
  │         └─ 사내 문서를 잘라서 벡터DB에 넣고 검색이 됐다
  │
  │   Part 2: 비서 만들기
  ├── Ch.4  비서가 말을 하기 시작했다  ★ 전환점
  │         └─ 검색 결과에 LLM을 연결했더니 자연어 답변이 나왔다
  │
  ├── Ch.5  비서에게 연장통을 쥐여주자
  │         └─ DB 조회까지 가능해졌다. 하나의 창구에서 모든 질문에 답한다
  │
  │   Part 3: 비서 키우기
  ├── Ch.6  비서에게 매뉴얼을 주자
  │         └─ 캐시·로그·설정. 체계 있는 비서가 됐다
  │
  └── Ch.7  비서의 실력을 올리자
            └─ RAG 파이프라인을 다듬어 답변 품질이 올라갔다
```

## 버전별 성과 — 챕터가 끝나면 무엇이 보이는가

| 챕터 | 버전 | 핵심 질문 | 끝나면 보이는 것 |
|------|------|----------|----------------|
| Ch.1 | v0.1 | 어디서부터 시작하나? | 브라우저에서 직원·매출 대시보드가 보인다 |
| Ch.2 | v0.2 | ChatGPT로 안 되는 이유는? | 실패 → 컨텍스트 → RAG, 3단계를 눈으로 확인한다 |
| Ch.3 | v0.3 | 문서를 어떻게 검색 가능하게 만드나? | CLI에서 "연차 규정" 검색하면 관련 문단이 나온다 |
| Ch.4 | v0.4 | 검색 결과를 어떻게 대화로 만드나? | 채팅 UI에서 질문하면 근거 + 답변이 나온다 |
| Ch.5 | v0.5 | DB 질문과 문서 질문을 어떻게 하나로? | "연차 며칠?"도 "보안 규정?"도 한 창구에서 답한다 |
| Ch.6 | v0.6 | 어떻게 제대로 된 구조로 만드나? | 데모 모드 실행, 로그 파일 확인, 캐시로 빠른 응답 |
| Ch.7 | v0.7 | 답변 품질을 어떻게 올리나? | Before/After 성능 비교표. 같은 질문에 더 정확한 답 |

## 이 책을 읽는 방법

**순서대로 읽으세요.** 각 챕터는 이전 챕터의 결과물 위에 만들어집니다. Ch.3의 벡터DB가 Ch.4의 RAG 체인 재료가 되고, Ch.4의 비서가 Ch.5에서 연장통을 받습니다.

**실행하면서 읽으세요.** 각 챕터 끝에 실습 코드가 있습니다. 이야기를 읽고 나서 코드를 실행하면 — "아, 그래서 이렇게 동작하는 거구나"를 직접 확인할 수 있습니다.

**완벽하게 이해하지 않아도 됩니다.** 처음엔 비유로 개념을 잡고, 기술 파트에서 정식 정의를 확인하는 구조입니다. 처음 읽을 때 모든 걸 이해하려 하지 않아도 됩니다. 전체 여정을 한 번 따라가는 것만으로도 RAG가 어떻게 돌아가는지 감이 생깁니다.
