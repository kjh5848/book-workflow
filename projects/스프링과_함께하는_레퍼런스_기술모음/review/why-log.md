## 2026-03-15-1 — 이미지 플레이스홀더 누락

**트리거**: 사용자 피드백 — "또 이미지 작업을 안했네? 왜 자꾸 이미지 작업을 안해?"

**대상**: CH01 Docker로 시작하는 개발 환경

**5 Whys 분석**

1. **왜 CH01에 이미지 플레이스홀더가 없는가?** → 작가 에이전트가 이미지 삽입을 하지 않았다
2. **왜 작가가 이미지를 삽입하지 않았는가?** → 챕터 작성 규칙에 이미지 플레이스홀더가 필수 항목으로 없었다
3. **왜 outline-v1.md에 이미지 배치 계획이 없는가?** → 뼈대 설계 시 이미지 위치를 명시하지 않았다
4. **왜 auto-chapters.md에 이미지 규칙이 없었는가?** → structure.md에 포맷 정의만 있고, 실제 삽입 강제 규칙이 없었다
5. **근본 원인**: 이미지 플레이스홀더가 "포맷 안내"로만 존재하고, "필수 삽입 규칙"으로 강제되지 않았다

**수정 내역**

- `.claude/rules/auto-chapters.md` writer `### 공통` 섹션에 규칙 1줄 추가:
  - "챕터를 이미지 플레이스홀더 없이 완성하지 마라. 실행 결과 → `[CAPTURE NEEDED]`, 아키텍처/흐름도 → `[GEMINI PROMPT]` 삽입 필수"
- CH01에 이미지 플레이스홀더 삽입 (docker-compose up 결과, localhost 접속, Docker 구조 개념도)

## 2026-03-15-2 — GEMINI PROMPT 베이스 스타일 미적용

**트리거**: 사용자 피드백 — "제미나이 이미지 프롬프트 넣을때 기본 미니멀리스트 규칙적용안해?"

**대상**: CH01 Docker 챕터의 GEMINI PROMPT 플레이스홀더

**5 Whys 분석**

1. **왜 GEMINI PROMPT가 베이스 스타일을 포함하지 않는가?** → visual 스킬의 image.md 형식을 참조하지 않았다
2. **왜 image.md를 참조하지 않았는가?** → 일러스트레이터 에이전트가 호출되지 않았다
3. **왜 일러스트레이터가 호출되지 않았는가?** → 메타코딩이 디스패치 순서를 건너뛰었다
4. **왜 건너뛰었는가?** → 디스패치 순서 준수를 강제하는 규칙이 없었다
5. **근본 원인**: 디스패치 테이블에 순서가 명시되어 있지만, "생략하지 마라"는 강제 규칙이 없어서 일러스트레이터를 스킵했다. 일러스트레이터가 호출되었다면 visual 스킬이 자동 적용되었을 것

**수정 내역**

- `.claude/agents/meta/AGENT.md` 운영 절차에 규칙 1줄 추가:
  - "디스패치 테이블의 에이전트 순서를 생략하지 마라. 특히 챕터 작성 시 일러스트레이터를 건너뛰지 마라"
- CH01의 GEMINI PROMPT를 image.md 형식(HTML 주석 + 베이스 스타일 + ../assets/ 경로 + 캡션)으로 수정

## 2026-03-15-3 — 레포 소개 방식 부실

**트리거**: 사용자 피드백 — "깃헙 레포는 주소로 넣는거 있지 말고 클론은 한다고 적어줘야 클론을해서 실행도 하고 완성된 코드는 어디에 있다고 설명도 해줘야지", "파일을 볼때 전체 파일구조를 제시하고 적어줘야지 작가야~~~ 파일계층 구조를 왜 안넣니?"

**대상**: CH01, CH02 레포 소개 섹션

**5 Whys 분석**

1. **왜 레포를 URL 테이블로만 나열했는가?** → 작가가 클론 명령/파일 트리/완성코드 안내를 작성하지 않았다
2. **왜 작성하지 않았는가?** → auto-chapters.md에 레포 소개 형식 규칙이 없었다
3. **왜 규칙이 없었는가?** → 기술 파트 시작 시 레포 소개를 어떤 형식으로 해야 하는지 정의하지 않았다
4. **근본 원인**: 기술 파트 시작 시 레포 소개 형식(git clone 명령 + 파일 계층 트리 + start/end 코드 안내)이 규칙화되지 않았다

**수정 내역**

- `.claude/rules/auto-chapters.md` writer에 `### 레포 소개` 섹션 추가:
  - "기술 파트 시작 시 레포 소개 블록 필수. URL 테이블만으로 끝내지 마라"
  - 형식: git clone 명령 → 파일 계층 트리([실습]/[설명]/[참고] 태그) → start/end 코드 안내
- CH01 레포 소개에 git clone 명령 + 파일 트리 추가
- CH02 레포 소개에 git clone 명령 + 파일 트리 + start/end 안내 추가
