# 씨앗 — 스프링과 함께하는 레퍼런스 기술 모음

## 의도 (한 문장)

> Spring Boot를 쓰는 주니어 개발자가 실무에서 마주치는 주변 기술들을 하나씩 직접 구현해보며 두려움을 없애는 책입니다.

## 독자

- **누구인가**: 1~2년차 주니어 백엔드 개발자
- **무엇을 알고 있는가**: Spring Boot로 REST API, CRUD, JPA 기본 사용. 간단한 웹 애플리케이션을 만들어본 경험
- **무엇을 모르는가**: Docker 컨테이너 운영, OAuth 소셜 로그인 연동, Redis 캐싱, 파일 업로드 아키텍처, 실시간 통신 방식, 검색 엔진, 메시지 큐. "들어는 봤지만 직접 해본 적은 없는" 기술들

## 책 유형

**독립형** — 챕터마다 서로 다른 기술을 다룹니다. 비유 전략도 챕터별로 각각 독립된 비유를 사용합니다. 기술이 다르면 비유도 달라야 합니다.

## 기술 범위

| 장 | 기술 | 핵심 키워드 |
|---|---|---|
| 1 | Docker | 컨테이너, 이미지, docker-compose |
| 2 | OAuth 2.0 / OIDC (Kakao) | 인가 코드, 액세스 토큰, 소셜 로그인 |
| 3 | Redis | 캐싱, 세션 저장, TTL |
| 4 | Base64 / S3 / Presigned URL / Lambda | 파일 업로드, 서버리스 이미지 처리 |
| 5 | Polling | 주기적 요청, 타이머, 상태 확인 |
| 6 | SSE (Server-Sent Events) | 단방향 스트림, 실시간 알림 |
| 7 | WebSocket | 양방향 통신, 실시간 채팅 |
| 8 | Elasticsearch | 전문 검색, 역인덱스, 한글 분석기 |
| 9 | RabbitMQ | 메시지 큐, 비동기 처리, Producer/Consumer |

## 톤

- 전체 존댓말 (-습니다/-입니다)
- 캐릭터 대화와 독백만 구어체
- "왜 이게 필요한가"를 이야기로 먼저 보여주고, 기술 파트에서 구현
- 설교하지 않고, 독자가 "나도 해봐야겠다"고 느끼게

## 이 책이 아닌 것

- 코드를 따라치는 튜토리얼이 아닙니다. 개념과 맥락을 잡는 책입니다.
- Spring Boot 입문서가 아닙니다. Spring Boot 기초는 이미 아는 독자를 대상으로 합니다.
- 운영 환경 구축 가이드가 아닙니다. 로컬에서 동작하는 수준까지 다룹니다.
- 깊이 있는 내부 구현 분석서가 아닙니다. "처음 써보는 사람이 전체 그림을 잡는" 수준입니다.
