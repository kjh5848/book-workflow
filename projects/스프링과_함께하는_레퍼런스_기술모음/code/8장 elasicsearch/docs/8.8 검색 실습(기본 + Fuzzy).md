Elasticsearch 실습은 아래 깃주소를 git clone 하고 직접 서버를 실행하여 실습합니다.

**실습 코드**

```java
https://github.com/metacoding-11-spring-reference/spring-elasticsearch
```

실습 전에 **8.7 오타 허용 검색(Fuzzy) 개념과 적용 기준**을 먼저 읽으면 흐름이 더 잘 잡힙니다.

이 실습은 **오타 검색만이 아니라 전체 검색 흐름**을 순서대로 확인합니다.

```text
실습 흐름
1) 실행 환경 준비
2) 데이터 저장
3) 기본 검색 확인
4) 오타(Fuzzy) 검색 확인
```

---

## **1) 실행 환경 준비**

---

### **1. Docker Compose 실행 (ES + Kibana + Spring app)**

```bash
cd elasticsearch
docker compose up -d
```

`docker compose up -d`로 ES/Kibana/Spring app이 함께 올라오며, Spring은 `http://localhost:8080`에서 동작합니다.
ES가 준비될 때까지 잠시 대기하세요.

---

### **2. Elasticsearch 응답 확인**

```bash
curl "http://localhost:9200"
```

클러스터 정보가 JSON으로 반환되면 ES가 정상 동작 중입니다.

---

### **3. Spring Boot 로컬 실행(선택)**

```bash
cd spring-elasticsearch
./gradlew bootRun
```

Docker Compose로 이미 실행 중이라면 이 단계는 건너뛰어도 됩니다.
로컬 실행을 할 경우 `app` 컨테이너를 내려 8080 포트 충돌을 피하세요.

로컬 실행 시에는 `src/main/resources/application.properties`의 주소를 아래처럼 맞춰야 합니다.

```properties
spring.elasticsearch.uris=http://localhost:9200
```

  -H "Content-Type: application/json" \

## **3) 기본 검색 동작 확인**

---

### **1. 정상 키워드 검색**

```bash
curl "http://localhost:8080/search?keyword=갤럭시"
```

정상 키워드로 검색했을 때 `갤럭시 S23`, `갤럭시 워치`가 나오면 기본 검색이 정상입니다.

---

## **4) 오타 허용 검색 요청**

---

**(확인) 경로: src/main/java/com/metacoding/spring_elasticsearch/ElasticSearch/ElasticSearchService.java**

### **1. Fuzzy 검색 메서드 확인**

```java
public List<DeviceEntity> searchAll(String keyword) {
    ...
    .multiMatch(m -> m
            .fields("title^3", "content")
            .query(keyword)
            .fuzziness("AUTO"))
    ...
}
```

---

### **2. 오타 검색 테스트**

```bash
curl "http://localhost:8080/search?keyword=갈럭시"
```

응답에 `갤럭시 S23`, `갤럭시 워치`가 나오면 Fuzzy가 정상 동작하는 것입니다.

---

## **5) 결과 비교 포인트**

---

- **정상 키워드 vs 오타 키워드**: 정상 검색과 유사한 결과가 나오면 Fuzzy 효과가 확인됩니다.
- **정확도**: 필요 이상으로 결과가 늘어나면 Fuzzy 적용 범위를 좁힙니다.
- **응답 지연**: 오타 허용은 비용이 있으니 응답 시간이 늘어나는지 확인합니다.

---

## **6) 결과가 안 나올 때 체크리스트**

---

- ES가 실행 중인지 확인: `http://localhost:9200`
- Spring 설정 확인: Docker 실행 시 `spring.elasticsearch.uris=http://elasticsearch:9200`, 로컬 실행 시 `http://localhost:9200`
- 데이터 저장이 정상인지 확인: `/devices` 저장 응답
