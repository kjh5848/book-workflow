## **1) Docker로 실습 환경 실행(로컬)**

---

**(확인) 경로: elasticsearch/docker-compose.yml**

### **Elasticsearch + Kibana + Spring 실행 구성**

```yaml
services:
  elasticsearch:
    image: docker.elastic.co/elasticsearch/elasticsearch:8.19.8
    ports:
      - "9200:9200"
    environment:
      - discovery.type=single-node
      - xpack.security.enabled=false
      - ES_JAVA_OPTS=-Xms1g -Xmx1g

  kibana:
    image: docker.elastic.co/kibana/kibana:8.19.8
    ports:
      - "5601:5601"

  app:
    build:
      context: .
      dockerfile: Dockerfile
    ports:
      - "8080:8080"

```

단일 노드 모드와 보안 OFF 설정이 없으면 로컬 실습에서 연결이 실패합니다. 실행은 `docker compose up -d`로 진행하며, 이때 Spring app도 함께 올라옵니다.

Elasticsearch를 도커로 실행하는 이유는 “Spring 서버 안에 내장된 기능”이 아니라, **별도의 검색 서버(프로세스)** 이기 때문입니다. 즉, Spring Boot를 실행한다고 해서 Elasticsearch가 같이 실행되지 않아서, 실습을 하려면 Elasticsearch를 먼저 켜줘야 합니다.

---

### **Spring은 어디서 실행하나요?**

기본은 Docker Compose에서 **Spring app을 함께 실행**하는 방식입니다.

**1) Docker Compose로 실행(권장)**

- `app` 서비스가 Spring Boot를 컨테이너로 실행합니다.
- ES/Kibana와 같은 네트워크에서 바로 통신합니다.
- 로컬 `spring-elasticsearch` 코드를 빌드해 실행하므로 컨테이너에서 별도 git clone이 필요 없습니다.

**2) 로컬(IDE/터미널)에서 실행(선택)**

- 디버깅이 필요할 때 선택합니다.
- 이 경우 `spring.elasticsearch.uris=http://localhost:9200`로 바꿔야 합니다.

즉, 기본은 Docker Compose로 함께 실행하고, 필요할 때만 로컬 실행으로 전환하면 됩니다.

## **2) Spring Boot 프로젝트 구성**

---

**(확인) 경로: spring-elasticsearch/build.gradle**

### **의존성 추가**

```java
//Spring Boot에서 Elasticsearch와 연동해 인덱싱·검색 같은 기능을 쉽게 구현할 수 있도록 Spring Data Elasticsearch를 포함하는 스타터 의존성입니다.
implementation 'org.springframework.boot:spring-boot-starter-data-elasticsearch'
```

Elasticsearch를 사용하기 위해 `spring-boot-starter-data-elasticsearch`를 추가합니다.

## **3) RDB(H2/MySQL) + Elasticsearch 동시 연결 설정**

---

**(확인) 경로: src/main/resources/application.properties**

### **H2 + Elasticsearch 연결 설정**

```java
spring.datasource.url=jdbc:h2:mem:testdb;MODE=MySQL
spring.datasource.username=sa
spring.datasource.password=

spring.elasticsearch.uris=http://elasticsearch:9200
```

로컬 PC에서 실행 시 `http://localhost:9200`을 사용할 수 있고, 도커 네트워크 안에서는 `http://elasticsearch:9200`으로 접근해야 합니다.
