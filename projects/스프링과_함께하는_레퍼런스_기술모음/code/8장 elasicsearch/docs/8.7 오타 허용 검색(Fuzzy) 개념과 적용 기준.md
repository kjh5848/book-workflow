## **1) Fuzzy는 토큰 단위 + 편집 거리로 동작**

---

Fuzzy는 **분석기(Analyzer)가 만든 토큰**을 기준으로 오타 허용을 판단합니다.
즉, 전체 문장이 아니라 **단어 단위**로 비교됩니다.

```text
입력: "갤럭시 워치"
토큰: ["갤럭시", "워치"]
```

오타 허용은 문자열을 **몇 번 수정하면 같은 단어가 되는지(편집 거리)** 를 기준으로 판단합니다.

```text
편집 거리 예시
갤럭시 → 갈럭시 (1회 교체)
아이폰 → 아이폰X (1회 추가)
```

편집 거리가 작을수록 “비슷한 단어”로 판단되어 검색 결과에 포함됩니다.

참고로 `keyword` 타입처럼 **분석되지 않는 필드**에서는 Fuzzy의 효과가 제한적이므로,
실습처럼 `Text` 필드에서 사용하는 것이 기본입니다.

## **2) AUTO 규칙 (길이 기준 자동 허용)**

---

`AUTO`는 단어 길이에 따라 허용 거리를 자동으로 결정합니다.

```text
길이 0~2  -> 허용 거리 0
길이 3~5  -> 허용 거리 1
길이 6+   -> 허용 거리 2
```

짧은 단어는 한 글자만 달라도 의미가 크게 변하므로, AUTO는 더 보수적으로 동작합니다.

---

## **3) 적용 기준과 주의점**

---

- **짧은 검색어는 제외**: 1~2글자는 오탐이 많아 Fuzzy 적용을 피합니다.
- **정확한 코드/모델명은 제외**: SKU, 모델번호 같은 값은 정확 매칭이 적합합니다.
- **필드 범위를 좁히기**: 제목처럼 중요한 필드에만 Fuzzy를 적용하면 품질이 올라갑니다.
- **성능 비용 인지**: 오타 허용은 후보 단어를 늘리므로 검색 비용이 증가합니다.

필요하면 `prefix_length`, `max_expansions` 같은 옵션으로 검색 범위를 줄이는 방법도 있습니다.
