# 8장 Elasticsearch 키워드 검색

## 8.1 왜 Elasticsearch가 필요한가

- RDB `LIKE %keyword%`의 한계
- 문서 내 단어 검색(Full-text search)
- 오타 허용(Fuzzy)
- 키워드 기반 검색(분석기/토큰화 개념만 최소)

---

## 8.2 실습 환경 준비

- Docker로 Elasticsearch 실행(로컬)
- Spring Boot 프로젝트 구성
- RDB(H2/MySQL) + Elasticsearch 동시 연결 설정

---

## 8.3 데이터 모델 설계 (RDB + ES)

- RDB Entity, ES Document
- “RDB PK를 ES 문서 식별자로 쓰는 이유”

---

## 8.4 Spring Data Elasticsearch 기본 사용법

- `@Document`, `@Field` 핵심 어노테이션
- ElasticsearchRepository vs 커스텀 Query
- 인덱스/매핑(최소만)

---

## 8.5 저장 흐름 구현: RDB 저장 + ES 동시 저장

- 서비스 계층 설계
- 트랜잭션 단위 처리 전략
    - RDB 저장 → ES 저장
    - 실패 시 롤백/보상 트레이드오프(간단 정리)
- 저장 API 구현 (POST)

---

## 8.6 검색 흐름 구현: ES 검색 → RDB 재조회

- 검색 API 설계 (keyword, page/size)
- ES 검색 쿼리 선택
    - match(문장 검색)
    - multi_match(필드 확장 시)
    - fuzziness(오타 허용)
- ES 결과에서 `sentenceId` 목록 추출
- RDB에서 `IN (...)`로 재조회 후 응답 조립

---

## 8.7 오타 허용 검색(Fuzzy) 개념과 적용 기준

- 토큰/분석기와 Fuzzy 관계
- 편집 거리 개념
- AUTO 규칙(길이 기준)
- 적용 기준과 주의점

---

## 8.8 검색 실습(기본 + Fuzzy)

- 데이터 저장
- 기본 검색 동작 확인
- 오타 검색 요청
- 결과 확인
