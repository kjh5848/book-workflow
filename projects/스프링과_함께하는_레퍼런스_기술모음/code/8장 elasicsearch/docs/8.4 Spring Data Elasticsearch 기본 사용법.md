## **1) 핵심 어노테이션 (@Document, @Field)**

---

**(확인) 경로: src/main/java/com/metacoding/spring_elasticsearch/device/DeviceDocument.java**

### **1. 인덱스 및 필드 매핑**

```java
@Document(indexName = "devices")
public class DeviceDocument {

    @Field(type = FieldType.Text)
    private String title;

    @Field(type = FieldType.Text)
    private String content;
}
```

`@Document`는 인덱스 이름을 지정하고, `@Field(type = Text)`는 분석기 기반 full-text 검색이 가능한 필드로 매핑합니다.

## **3) 커스텀 Query (ElasticsearchOperations)**

---

**(확인) 경로: src/main/java/com/metacoding/spring_elasticsearch/ElasticSearch/ElasticSearchService.java**

### **1. NativeQuery 기반 검색**

```java
public List<DeviceEntity> searchAll(String keyword) {
    NativeQuery query = NativeQuery.builder()
            .withQuery(q -> q.bool(b -> b
                    .should(s -> s.multiMatch(m -> m
                            .fields("title^3", "content")
                            .query(keyword)
                            .fuzziness("AUTO")))
                    .minimumShouldMatch("1")))
            .build();

    var deviceHits = operations.search(query, DeviceDocument.class);
    ...
}
```

`ElasticsearchOperations`를 사용하면 DSL 수준의 검색 쿼리를 직접 구성할 수 있어 복잡한 검색 조건을 구현하기 쉽습니다.
