# 5.7 Postman 실습 2: 완료 저장 요청으로 저장 흐름 검증

> 목표: 업로드 후 5초 대기한 뒤 완료 저장 API를 호출해 DB 저장을 검증합니다.

## 사전 조건
- 5.4 단계에서 업로드를 완료한 뒤 5초 대기합니다.

## 1) 완료 저장 요청
- Method는 `POST`입니다.
- URL은 `http://localhost:8080/api/images/complete`입니다.
- Headers는 `Content-Type: application/json`입니다.

(입력) 경로: postman/5.7-complete-request.json
```json
{
  "fileName": "sample.jpg",
  "key": "original/uuid.jpg"
}
```
완료 저장 요청 바디 예시입니다.

## 2) 응답 확인
- 응답에 `id`, `uuid`, `originalUrl`, `resizedUrl`가 포함되는지 확인합니다.

(확인) 경로: postman/5.7-complete-response.json
```json
{
  "id": 1,
  "uuid": "uuid",
  "originalUrl": "https://bucket.s3.region.amazonaws.com/original/uuid.jpg",
  "resizedUrl": "https://bucket.s3.region.amazonaws.com/resized/uuid.jpg",
  "fileName": "sample.jpg",
  "createdAt": "2025-01-01 12:00"
}
```
저장 응답 형식을 확인합니다.

## 3) H2 콘솔 확인
- `http://localhost:8080/h2-console`에 접속합니다.
- `image_tb` 테이블에 데이터가 저장되었는지 확인합니다.
