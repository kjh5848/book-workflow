# 5.1 Quick Start: 환경변수 수정 → 실행 → Postman으로 전체 흐름 확인

> 목표: .env 값만 바꾸고 실행한 뒤, Postman으로 presigned 발급 → S3 업로드 → 5초 대기 → 완료 저장 → 조회 흐름을 확인합니다.

## 준비물
- AWS 계정, S3 버킷, IAM Access Key/Secret Key가 필요합니다.
- Java 21, Gradle이 필요합니다.
- Postman이 필요합니다.

## 1) AWS 기본 설정
[이미지]
- S3 버킷을 생성합니다 (리전: ap-northeast-2 등, 퍼블릭 액세스 차단 유지).

[이미지]
- IAM 사용자를 생성하고 Access Key를 발급합니다.
- 권한은 S3 PutObject/GetObject로 최소 범위를 부여합니다.

## 2) 환경변수 작성
> 실습에서는 .env만 수정하는 것을 원칙으로 합니다. 실제 실행 시에는 .env 값을 application.properties에 반영합니다.

(입력) 경로: .env
```env
AWS_REGION=ap-northeast-2
AWS_S3_BUCKET=your-bucket-name
AWS_ACCESS_KEY=YOUR_ACCESS_KEY
AWS_SECRET_KEY=YOUR_SECRET_KEY
```
이 파일에 실습에 필요한 AWS/Spring 환경변수를 정의합니다.

## 3) Spring 실행
- 실행 전 `.env`를 환경변수로 주입합니다 (예: `set -a; source .env; set +a`).
- `spring-presign-url` 디렉터리에서 `./gradlew bootRun`을 실행합니다.
- 서버 주소는 `http://localhost:8080`입니다.

> Lambda는 로컬호스트로 웹훅을 보낼 수 없으므로, 업로드 후 5초 대기한 뒤 클라이언트가 완료 저장 API를 호출합니다.

## 4) Postman으로 흐름 확인
- 5.4에서 업로드 후 5초 대기합니다.
- 5.7, 5.9의 실습 순서대로 요청을 실행합니다.
- 성공 기준은 다음과 같습니다.
  - Presigned URL 발급 OK
  - S3 PUT 업로드 OK
  - 완료 저장 OK
  - 목록/상세 조회 OK
