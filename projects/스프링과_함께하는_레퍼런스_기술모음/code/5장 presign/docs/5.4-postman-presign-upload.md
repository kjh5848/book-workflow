# 5.4 Postman 실습 1: Presigned 발급 → S3에 PUT 업로드 → 업로드 확인

> 목표: Postman으로 presigned URL을 발급받고, 해당 URL로 S3에 PUT 업로드합니다.

## 1) Presigned 발급 요청
- Method는 `POST`입니다.
- URL은 `http://localhost:8080/api/images/presigned`입니다.
- Headers는 `Content-Type: application/json`입니다.

(입력) 경로: postman/5.4-presign-request.json
```json
{
  "fileName": "sample.jpg",
  "contentType": "image/jpeg"
}
```
Presigned 발급 요청 바디 예시입니다.

## 2) Presigned 응답 확인
- 응답에서 `uploadUrl`, `objectKey`를 확인합니다.

(확인) 경로: postman/5.4-presign-response.json
```json
{
  "uploadUrl": "https://...",
  "objectKey": "original/uuid.jpg",
  "expiresAt": "2025-01-01T00:00:00Z"
}
```
Presigned 응답 형식을 확인합니다.

## 3) S3 PUT 업로드
- Method는 `PUT`입니다.
- URL은 `{{uploadUrl}}`입니다.
- Headers는 `Content-Type: image/jpeg`입니다.
- Body는 `binary`로 파일을 선택합니다.

## 4) S3 업로드 확인
[이미지]
- S3 콘솔에서 `original/` 경로에 업로드된 객체를 확인합니다.
