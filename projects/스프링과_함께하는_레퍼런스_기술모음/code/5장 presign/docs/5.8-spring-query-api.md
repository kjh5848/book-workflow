# 5.8 Spring 구현 3: 목록/상세 조회 API(응답 표준 + URL 전략)

> 목표: 저장된 이미지 메타를 목록/상세로 조회합니다.

## 1) 서비스에 조회 메서드 추가

(입력) 경로: src/main/java/com/metacoding/spring_presign_url/image/ImageService.java
```java
public ImageResponse.Items listAll() {
    java.util.List<ImageResponse.Item> items = imageRepository.findAll()
            .stream()
            .map(ImageResponse.Item::fromEntity)
            .toList();
    return new ImageResponse.Items(items);
}

public ImageResponse.Detail findById(Long id) {
    ImageEntity entity = imageRepository.findById(id)
            .orElseThrow(() -> new RuntimeException("이미지를 찾을 수 없습니다."));
    return ImageResponse.Detail.fromEntity(entity);
}
```
전체 조회와 단건 조회 서비스를 제공합니다.

## 2) 컨트롤러에 목록/상세 엔드포인트 추가

(입력) 경로: src/main/java/com/metacoding/spring_presign_url/image/ImageController.java
```java
@org.springframework.web.bind.annotation.GetMapping("/list")
public ImageResponse.Items getAllImages() {
    return imageService.listAll();
}

@org.springframework.web.bind.annotation.GetMapping("/{id}")
public ImageResponse.Detail getImageDetail(@org.springframework.web.bind.annotation.PathVariable Long id) {
    return imageService.findById(id);
}
```
목록/상세 API 엔드포인트를 연결합니다.

## 3) 응답 DTO 표준화(Items/Detail)

목록과 상세 응답을 동일한 포맷으로 감싸면 클라이언트에서 파싱이 쉬워지고, 이후 메타 정보(페이징 등)를 확장하기 좋습니다.

(입력) 경로: src/main/java/com/metacoding/spring_presign_url/image/ImageResponse.java
```java
public record Item(Long id, String originalUrl, String resizedUrl) {
}

public record Items(java.util.List<Item> items) {
}

public record Detail(
        Long id,
        String uuid,
        String originalUrl,
        String resizedUrl,
        String fileName,
        @com.fasterxml.jackson.annotation.JsonFormat(pattern = "yyyy-MM-dd HH:mm")
        java.time.LocalDateTime createdAt) {
}
```
목록은 `items`, 상세는 `detail`로 고정합니다. 목록은 가벼운 필드만 반환합니다.

(변경) 경로: src/main/java/com/metacoding/spring_presign_url/image/ImageController.java
```java
@org.springframework.web.bind.annotation.GetMapping("/list")
public ImageResponse.Items getAllImages() {
    return imageService.listAll();
}

@org.springframework.web.bind.annotation.GetMapping("/{id}")
public ImageResponse.Detail getImageDetail(@org.springframework.web.bind.annotation.PathVariable Long id) {
    return imageService.findById(id);
}
```
응답을 `Items/Detail`로 감쌉니다.

예시 응답:
```json
{
  "items": [
    {
      "id": 1,
      "originalUrl": "https://bucket.s3.region.amazonaws.com/original/uuid.jpg",
      "resizedUrl": "https://bucket.s3.region.amazonaws.com/resized/uuid.jpg"
    }
  ]
}
```

```json
{
  "detail": {
    "id": 1,
    "uuid": "uuid",
    "originalUrl": "https://bucket.s3.region.amazonaws.com/original/uuid.jpg",
    "resizedUrl": "https://bucket.s3.region.amazonaws.com/resized/uuid.jpg",
    "fileName": "sample.jpg",
    "createdAt": "2025-01-01 12:00"
  }
}
```

## 4) URL 전략
- 서버는 `original/{uuid}` 키로 `resized/{uuid}.jpg` URL을 조합해 저장합니다.
- S3 버킷이 비공개인 경우 Presigned GET을 추가 발급하는 방식으로 확장합니다.
- 실습은 공개 URL 형태를 그대로 반환합니다.
