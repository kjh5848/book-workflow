# 5.5 Lambda 구현: S3 이벤트 수신 → 이미지 처리 → 결과 업로드

> 목표: S3 ObjectCreated 이벤트로 Lambda를 실행하고, 리사이즈된 이미지를 `resized/` 경로에 저장합니다.

## 1) Lambda 생성 및 트리거 연결
[이미지]
- 런타임은 Node.js 18.x입니다.
- 메모리는 512MB 이상을 권장합니다.
- 타임아웃은 10초 이상을 권장합니다.

[이미지]
- S3 트리거를 추가합니다.
- 이벤트는 `ObjectCreated:Put`입니다.
- Prefix는 `original/`입니다.

## 2) 환경변수 설정
[이미지]
- 이 실습에서는 별도 환경변수를 사용하지 않습니다.

## 3) Lambda 코드

(입력) 경로: lambda/package.json
```json
{
  "name": "image-resizer",
  "version": "1.0.0",
  "type": "commonjs",
  "dependencies": {
    "aws-sdk": "^2.1571.0",
    "sharp": "^0.33.4"
  }
}
```
Lambda에서 사용하는 의존성을 정의합니다.

(입력) 경로: lambda/index.js
```javascript
const AWS = require("aws-sdk");
const sharp = require("sharp");

const s3 = new AWS.S3();
const OUTPUT_PREFIX = "resized/";

exports.handler = async (event) => {
  const record = event.Records?.[0];
  if (!record) return;

  const bucket = record.s3.bucket.name;
  const key = decodeURIComponent(record.s3.object.key.replace(/\+/g, " "));

  if (!key.startsWith("original/")) return;

  const original = await s3.getObject({ Bucket: bucket, Key: key }).promise();
  const resizedBuffer = await sharp(original.Body)
    .resize({ width: 600 })
    .jpeg({ quality: 80 })
    .toBuffer();

  const resizedKey = key
    .replace("original/", OUTPUT_PREFIX)
    .replace(/\.[^/.]+$/, ".jpg");

  await s3.putObject({
    Bucket: bucket,
    Key: resizedKey,
    Body: resizedBuffer,
    ContentType: "image/jpeg"
  }).promise();

  return;
};
```
S3 이벤트 수신 후 리사이즈와 업로드 흐름을 구현합니다.
Lambda는 서버로 웹훅을 보내지 않으며, 결과는 S3 `resized/` 경로에만 저장합니다.

## 4) 배포
- `lambda` 디렉터리에서 `npm install`을 실행합니다.
- `lambda` 전체를 zip으로 압축해 업로드합니다.
- 업로드 후 테스트 이벤트로 실행을 확인합니다.
