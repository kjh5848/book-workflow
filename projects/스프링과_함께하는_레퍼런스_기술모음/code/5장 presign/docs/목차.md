# 5장 Presigned URL 기반 사진 업로드 + S3 이벤트 + Lambda 처리 + Spring 메타 저장 (Postman 실습)

> 이 장 목표: 독자는 **환경변수(.env)만 수정**하고 `docker compose up` 후, **Postman 컬렉션 실행**만으로 업로드→처리→조회 흐름을 재현합니다.

---

## 5.1 Quick Start: 환경변수 수정 → 실행 → Postman으로 전체 흐름 확인 ([문서](5.1-quick-start.md))
- **독자 수정 포인트는 `.env` 1개 파일**로 고정합니다(버킷명/리전/AWS 키 등).
- `docker compose up -d`로 Spring(H2 포함) 실행 후 Postman 컬렉션을 바로 돌립니다.
- 성공 기준: presigned 발급 OK → S3 PUT 업로드 OK → 5초 대기 후 완료 저장 OK → 조회 OK.

## 5.2 전체 아키텍처와 규칙 고정하기(키 규칙/상태/권한) ([문서](5.2-architecture-rules.md))
- 구성요소: Spring(API) / S3 / Lambda / H2 / Postman
- **S3 Object Key 규칙**(original/resized 경로, 파일명 규칙)을 먼저 확정합니다.
- **상태 모델**(UPLOADED/PROCESSED/FAILED)과 **권한 모델**(Presigned PUT, Lambda 콜백 없음)을 한 장에 정리합니다.

## 5.3 Spring 구현 1: Presigned URL 발급 API(업로드 제약 포함) ([문서](5.3-spring-presign-api.md))
- 요청 DTO: fileName, contentType, size(optional)
- 발급 로직: 만료시간, key 생성 규칙, contentType 제한, size 제한(가능하면)
- 응답 DTO: uploadUrl, objectKey, expiresAt

## 5.4 Postman 실습 1: Presigned 발급 → S3에 PUT 업로드 → 업로드 확인 ([문서](5.4-postman-presign-upload.md))
- Postman으로 presigned URL 발급 요청
- 발급받은 uploadUrl로 **PUT 업로드**
- S3 콘솔에서 객체 생성 여부로 1차 성공 확인

## 5.5 Lambda 구현: S3 이벤트 수신 → 이미지 처리 → 결과 업로드 ([문서](5.5-lambda-image-process.md))
- S3 ObjectCreated 이벤트 파싱(bucket/key)
- 이미지 처리: 리사이즈/압축/포맷 변환(옵션 범위 명확히)
- 결과 저장: `resized/` 규칙에 맞춰 업로드 + 메타(width/height/size/ext) 구성

## 5.6 Spring 구현 2: 완료 저장 API(요청/응답/저장) ([문서](5.6-spring-callback.md))
- 클라이언트가 완료 저장 요청을 호출합니다.
- 요청 DTO: fileName, key
- 리사이즈 파일 존재를 1회 확인한 뒤 URL을 조합해 저장합니다.

## 5.7 Postman 실습 2: 완료 저장 요청으로 저장 흐름 검증 ([문서](5.7-postman-callback-sim.md))
- 업로드 후 5초 대기한 뒤 완료 저장 API를 호출합니다.
- 성공 기준: H2 콘솔/조회 API에서 데이터가 보입니다.

## 5.8 Spring 구현 3: 목록/상세 조회 API(응답 표준 + URL 전략) ([문서](5.8-spring-query-api.md))
- 목록 API: uuid, fileName, size, ext, status, createdAt
- 상세 API: uuid로 조회 → originalUrl/resizedUrl 반환
- URL 전략: S3 퍼블릭이 아닌 경우 Presigned GET 발급 여부를 옵션으로 명시

## 5.9 Postman 실습 3: 목록 → 상세 → URL 열람으로 최종 확인 ([문서](5.9-postman-list-detail.md))
- 목록 조회로 uuid 확보
- 상세 조회로 URL 확인
- 브라우저/포스트맨으로 이미지 열어 최종 성공 체크

## 5.10 React 연동하기(프리사인 업로드 + 미리보기 + 저장) ([문서](5.10-react-integration.md))
- `react-presign-url`로 Spring API 연동
- 미리보기와 기본 이미지(캡쳐본) 적용
- CORS 설정 및 localStorage 개념 정리

## 5.11 트러블슈팅 체크리스트(403/트리거/완료/권한) ([문서](5.11-troubleshooting.md))
- Presigned 403(SignatureMismatch, 만료, Content-Type 불일치)
- 업로드 성공인데 Lambda 미동작(이벤트 설정/권한)
- Lambda 처리 실패(메모리/타임아웃/라이브러리)
- 완료 저장 실패(5초 대기/리사이즈 객체 확인)
- URL 접근 실패(퍼블릭 차단/Presigned GET 필요)
