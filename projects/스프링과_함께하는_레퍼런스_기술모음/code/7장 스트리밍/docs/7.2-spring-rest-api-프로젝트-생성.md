## **7.2 Spring REST API 프로젝트 생성**

---

**(확인) 경로: rtmp/build.gradle**

### **1. 의존성 설정(FFmpeg, Web, Mustache)**

```gradle
dependencies {
    implementation 'org.springframework.boot:spring-boot-starter-web'
    implementation 'org.springframework.boot:spring-boot-starter-mustache'
    implementation 'org.springframework.boot:spring-boot-starter-data-jpa'
    runtimeOnly 'com.h2database:h2'
    implementation("net.bramp.ffmpeg:ffmpeg:0.8.0")
}
```

초보자가 헷갈리기 쉬운 포인트만 간단히 정리하면 아래와 같습니다.

- `spring-boot-starter-web`: REST API를 만들기 위한 기본 기능(컨트롤러, JSON 처리 등)
- `spring-boot-starter-mustache`: 간단한 HTML 템플릿 페이지 제공용
- `spring-boot-starter-data-jpa` + `h2`: 샘플 프로젝트 기본 구성(이번 장에서는 직접 쓰지 않아도 포함됨)
- `net.bramp.ffmpeg:ffmpeg`: **FFmpeg 명령을 자바 코드로 만들게 해주는 래퍼**

**중요:** 이 라이브러리는 “FFmpeg 프로그램 자체”가 아닙니다. 실제 인코딩은 **내 PC에 설치된 FFmpeg 바이너리**가 실행합니다. 따라서 로컬에 FFmpeg가 설치되어 있고, PATH에 잡혀 있어야 정상 동작합니다.

---

**(확인) 경로: rtmp/src/main/resources/application.properties**

### **2. 업로드 용량/서버 설정**

```properties
server.port=8080
spring.servlet.multipart.max-file-size=2GB
spring.servlet.multipart.max-request-size=2GB
```

- `server.port=8080`: 서버가 열리는 포트입니다. (브라우저에서 `localhost:8080`으로 접근)
- `max-file-size`: **파일 하나의 최대 크기** 제한입니다.
- `max-request-size`: **요청 전체 크기**(파일 + 기타 필드) 제한입니다.

대용량 업로드를 위해 두 값을 2GB로 확장합니다.

**(확인) 경로: rtmp/src/main/java/com/metacoding/hls/service/HlsService.java**

### **3. 로컬 저장 경로 규칙**

```java
private final String ROOT_DIR = System.getProperty("user.dir") + "/upload/";
private final String ORIGINAL_DIR = ROOT_DIR + "original/";
private final String CONVERT_DIR_720 = ROOT_DIR + "hls/720p/";
private final String CONVERT_DIR_1080 = ROOT_DIR + "hls/1080p/";
```

경로가 실제로 어떻게 만들어지는지 예시로 보면 이해가 쉽습니다.

- `user.dir`는 **프로젝트 루트 경로**입니다.  
  예: `/Users/nomadlab/Desktop/workspace/.../7장 스트리밍`
- `ROOT_DIR`는 그 아래 `upload/` 폴더입니다.
- `ORIGINAL_DIR`는 원본 파일이 저장되는 폴더입니다.  
  예: `.../upload/original/video.mp4`
- `CONVERT_DIR_720`, `CONVERT_DIR_1080`는 변환된 HLS 결과가 저장되는 폴더입니다.  
  예: `.../upload/hls/720p/video.m3u8`

원본과 HLS 결과를 분리 저장하여, 품질별 디렉토리로 바로 접근할 수 있게 구성합니다.

**(확인) 경로: rtmp/src/main/java/com/metacoding/hls/controller/HlsController.java**

### **4. 접근 URL 규칙**

```java
@PostMapping("/upload")
@GetMapping("/hls/{quality}/{fileName}.m3u8")
@GetMapping("/hls/{quality}/{tsSegment}.ts")
```

라우팅 규칙을 실제 URL 예시로 보면 더 직관적입니다.

- 업로드: `POST /upload`  
  예: `http://localhost:8080/upload`
- HLS 재생 목록(m3u8):  
  예: `http://localhost:8080/hls/720p/video.m3u8`
- HLS 세그먼트(ts):  
  예: `http://localhost:8080/hls/720p/video0.ts`

`quality`는 `720p`/`1080p`처럼 해상도를 의미하고, `fileName`은 현재 프로젝트에서는 `video`로 고정됩니다.

---

**(확인) 경로: rtmp/src/main/java/com/metacoding/hls/HlsApplication.java**

### **5. 비동기 활성화(@EnableAsync)**

```java
@SpringBootApplication
@EnableAsync
public class HlsApplication {
    public static void main(String[] args) {
        SpringApplication.run(HlsApplication.class, args);
    }
}
```

`@EnableAsync`가 있어야 `@Async` 메서드가 **별도 스레드에서 동작**합니다.  
이 설정이 없으면 비동기 메서드도 결국 동기처럼 실행됩니다.
