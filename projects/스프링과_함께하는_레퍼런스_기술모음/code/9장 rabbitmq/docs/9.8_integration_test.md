## **1) 실행 순서**

메시지 기반 실습은 “수신자가 먼저 떠 있어야” 로그가 깔끔합니다.

1. RabbitMQ 실행(docker compose up)
2. Consumer ./gradlew bootrun실행(큐 리스닝 시작)
3. Producer  ./gradlew bootrun실행(폴링 시작)
4. GitHub의 README.md 변경 → 커밋/푸시
5. Consumer 로그에서 수신/반영 확인
