## 3) 의존성 준비

Producer와 Consumer는 모두 RabbitMQ 브로커와 AMQP 프로토콜로 통신합니다. Spring Boot에서는 AMQP 연동을 위해 `spring-boot-starter-amqp` 의존성이 필요합니다. 이 의존성 하나로 RabbitTemplate(발행), @RabbitListener(수신), ConnectionFactory 등 RabbitMQ 통신에 필요한 구성 요소가 기본으로 포함됩니다.

### 3-1. Producer 의존성 추가

Producer는 README.md 변경을 감지한 뒤 메시지를 RabbitMQ로 발행해야 하므로, AMQP Starter가 필요합니다.

**(확인) 경로: rabbitmq-producer/build.gradle**

```
dependencies {
    implementation 'org.springframework.boot:spring-boot-starter-amqp'
    // 기타 의존성...
}

```

### 3-2. Consumer 의존성 추가

Consumer는 RabbitMQ Queue를 구독하고 메시지를 수신하여 README.md를 반영해야 하므로, 동일하게 AMQP Starter가 필요합니다.

**(확인) 경로: rabbitmq-consumer/build.gradle**

```
dependencies {
    implementation 'org.springframework.boot:spring-boot-starter-amqp'
    // 기타 의존성...
}

```

### 3-3. 왜 Producer/Consumer 둘 다 추가해야 하나

- Producer는 `RabbitTemplate` 등을 사용해 **메시지를 발행(publish)** 해야 합니다.
- Consumer는 `@RabbitListener` 등을 사용해 **큐를 구독(consume)** 하고 메시지를 받아야 합니다.
- 따라서 둘 중 하나만 추가하면 한쪽은 정상 동작하더라도, 다른 한쪽은 RabbitMQ 연동 코드가 동작하지 않습니다.

## **4) GitHub 준비(**Producer**가 볼 대상)**

Observer는 README.md의 변경 여부를 확인해야 하므로, 저장소/경로/권한이 맞아야 합니다.

### **4-1. 설정 파일 위치(**Producer**)**

**(확인) 경로: rabbitmq-Producer/src/main/resources/application.properties**

### **4-2. GitHub 설정값**

```java
# GitHub 저장소 소유자(계정 또는 조직)
github.owner=자신의 깃헙 계정!!

# 저장소 이름
github.repo=config-readme

# 감지 대상 파일 경로(리포지토리 루트 기준)
github.readme-path=README.md

# 폴링 주기(ms) - 60000ms = 60초
github.poll-interval-ms=60000
```

### **4-3. README.md 경로 고정이 중요한 이유**

github.readme-path가 틀리면 Producer가 엉뚱한 파일을 보게 됩니다. 실습 단계에서는 README.md를 고정하고, 이후 확장(다른 파일)로 넘어가는 것이 디버깅이 쉽습니다.

## **5) 메시징 토폴로지 설정(Producer/Consumer 공통)**

이 단계에서 가장 많이 발생하는 문제는 “이름 불일치”입니다. Producer와 Consumer가 **같은 Exchange/Queue/Routing Key를 바라보도록** 먼저 통일합니다.

### **5-1. Producer 설정**

**(확인) 경로: rabbitmq-Producer/src/main/resources/application.properties**

```java
# ===== RabbitMQ 브로커 연결 설정 =====
spring.rabbitmq.host=localhost
spring.rabbitmq.port=5672
spring.rabbitmq.username=guest
spring.rabbitmq.password=guest

# ===== 메시징 토폴로지 설정(공통) =====
rabbit.exchange=github.events
rabbit.queue=repo-updates
rabbit.routing-key=readme.changed
```

---

### **5-2. Consumer 설정**

**(확인) 경로: rabbitmq-consumer/src/main/resources/application.properties**

```java
spring.rabbitmq.host=localhost
spring.rabbitmq.port=5672
spring.rabbitmq.username=guest
spring.rabbitmq.password=guest

# Consumer는 Producer와 동일한 queue 이름을 사용해야 한다
rabbit.queue=repo-updates
```
