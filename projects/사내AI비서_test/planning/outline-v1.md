## Part 1: LLM의 한계와 RAG의 필요성

> 신입 개발자가 "사내 AI 비서 만들어봐"라는 미션을 받고,
> LLM의 환각을 체험한 뒤 RAG로 문제를 해결하는 여정.

---

### Ch.1: AI에게 물어봤더니 거짓말을 한다 — v0.1

**핵심 개념**: LLM, 환각(Hallucination), 로컬 LLM
**기술**: Ollama, ChatOllama, DeepSeek R1:8b
**버전 성과**: 터미널에서 LLM에게 질문 → 환각 답변 출력 (실패 체험)
**예상 분량**: ~8p

**이야기 파트**:
- 도입: 팀장이 "회사 규정 물어보면 답해주는 AI 만들어봐"라고 미션을 던짐
- 개념: LLM이란? (비유: "세상의 모든 책을 읽은 박사 — 근데 우리 회사 내부 문서는 못 읽었다")
- 시나리오: LLM에게 "우리 회사 연차 규정 알려줘" → 자신있게 엉터리 답변
- 닫기: "이 녀석은 모르면 모른다고 안 해... 어떡하지?"

**코드 실습 분류**:
```
v0.1/
└── main.py    [실습] LLM 연결 + 질문 → 환각 답변 체험
```

**실습 요약**: 실습 1개, 설명 0개, 참고 0개

### Ch.2: 정보를 알려주면 되지 않을까? — v0.2

**핵심 개념**: Context Injection, 프롬프트 구성
**기술**: f-string 프롬프트, ChatOllama
**버전 성과**: 같은 질문에 정확한 답변 출력 (프롬프트에 정보 직접 삽입)
**예상 분량**: ~8p

**이야기 파트**:
- 도입: "모르면 알려주면 되지!" — 직관적 해결책
- 개념: Context Injection이란? (비유: "시험 보는데 컨닝페이퍼를 붙여준 셈")
- 시나리오: 프롬프트에 회사 규정을 넣어서 질문 → 정확한 답변!
- 닫기: "근데... 문서가 1000개면? 프롬프트에 다 못 넣잖아"

**코드 실습 분류**:
```
v0.2/
└── main.py    [실습] 프롬프트에 context 삽입 → 정확한 답변 체험
```

**실습 요약**: 실습 1개, 설명 0개, 참고 0개

### Ch.3: 문서를 통째로 넣으면 안 될까? — v0.3

**핵심 개념**: 벡터 DB, 임베딩, RAG 기초, 청킹의 부재
**기술**: ChromaDB, OllamaEmbeddings, nomic-embed-text, RetrievalQA
**버전 성과**: RAG가 작동하지만 불필요한 규정까지 다 나옴 (비효율 체감)
**예상 분량**: ~12p

**이야기 파트**:
- 도입: "프롬프트에 다 못 넣으니까, 필요한 것만 찾아서 넣어주는 건?" — RAG 아이디어 등장
- 개념: 임베딩이란? (비유: "문장을 좌표로 바꿔서 비슷한 것끼리 가까이 놓는 것")
- 개념: 벡터 DB란? (비유: "AI 전용 도서관 색인 — 의미로 검색")
- 시나리오: 문서를 통째로 벡터 DB에 넣고 RAG 실행 → 되긴 하는데 보안규정, 복지규정까지 다 나옴
- 닫기: "왜 휴가 물어봤는데 보안 규정까지 나오지? 뭔가 빠졌다..."

**코드 실습 분류**:
```
v0.3/
└── main.py    [실습] 통째 Document → ChromaDB → RAG 실행 → 비효율 체감
```

**실습 요약**: 실습 1개, 설명 0개, 참고 0개

### Ch.4: 잘게 쪼개니까 달라졌다 — v0.4

**핵심 개념**: 청킹(Chunking), Retriever, RAG 파이프라인
**기술**: Document 개별 분리, as_retriever(), RetrievalQA
**버전 성과**: 관련 규정만 검색되어 깔끔한 답변 (RAG 성공!)
**예상 분량**: ~10p

**이야기 파트**:
- 도입: "도서관에서 책 한 권 통째로 빌리는 것 vs 필요한 페이지만 복사하는 것"
- 개념: 청킹이란? (비유: "큰 문서를 뜻이 통하는 단위로 잘게 쪼개는 것")
- 시나리오: 규정을 인사/보안/복지로 쪼개서 넣음 → 휴가 질문에 인사규정만 나옴!
- 닫기: "드디어 제대로 된 AI 비서가 나왔다! 근데 하나 더..."

**코드 실습 분류**:
```
v0.4/
└── main.py    [실습] 개별 Document → 청킹된 RAG → 정확한 검색 체험
```

**실습 요약**: 실습 1개, 설명 0개, 참고 0개

---

### Ch.5: 검색을 넘어 생각까지 — v0.5

**핵심 개념**: RAG + 추론(Reasoning), Chain-of-Thought
**기술**: DeepSeek R1 추론 능력, 복합 질문 처리
**버전 성과**: "리프레시 데이 몇 번 남았어?" → 규정 검색 + 계산까지 답변
**예상 분량**: ~10p

**이야기 파트**:
- 도입: "검색은 잘 되는데... '몇 번 남았어?'는 계산을 해야 하잖아"
- 개념: 추론이란? (비유: "검색은 도서관 사서, 추론은 회계사 — 둘 다 필요할 때가 있다")
- 시나리오: "입사 6개월차, 리프레시 데이 2번 썼는데 몇 번 남았어?" → 규정 검색 + 계산 → 정확한 답변
- 닫기: "검색도 되고, 생각도 하는 AI 비서. 이제 진짜 쓸만하다!"

**코드 실습 분류**:
```
v0.5/
└── main.py    [실습] 복합 질문 → RAG + 추론 → 계산 답변 체험
```

**실습 요약**: 실습 1개, 설명 0개, 참고 0개
