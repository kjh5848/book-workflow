# 씨앗

## 주제
사내 AI 비서 만들기 — RAG + MCP를 활용해 사내 문서 기반 AI 업무 비서를 처음부터 끝까지 구축

## 집필 동기
실제로 사내에 AI 비서를 도입해본 경험을 공유하고 싶다. 동시에 AI/RAG를 모르는 개발자도 따라할 수 있는 쉬운 가이드가 필요하다고 느꼈다.

## 저자 경험
(스킵)

## 핵심 메시지
"사내 AI, 직접 만들 수 있다" — 외부 솔루션 없이도 우리 회사에 맞는 AI 비서를 구축할 수 있다.

## 책의 의도
독자가 RAG + MCP까지 연결해서 완성된 사내 AI 비서를 직접 구축할 수 있게 한다. "이해"에서 그치지 않고 "만들 수 있다"가 목표.

## 대상 독자
개발은 할 줄 아는데 AI/RAG만 모르는 개발자. Python 기본 문법, API 호출, DB 개념 정도는 알고 있다고 가정한다.

## 의도 밖 범위
- LangChain 내부 구조 (프레임워크 자체를 깊이 파는 것)
- 프로덕션 배포 (Docker, 클라우드, 모니터링, 스케일링)
- 보안/인증 (API 키 관리, 사용자 인증, 권한 제어)
- MCP 프로토콜 심화 (MCP 스펙, 커스텀 MCP 서버 개발)
- 프롬프트 엔지니어링 심화 (고급 프롬프트 기법, Few-shot 등)
- 다른 LLM 비교 (GPT, Claude 등과의 벤치마크)

## 차별점
- 사내 도입 실경험 기반
- 실패에서 시작하는 스토리텔링 (환각 체험 → 왜 RAG가 필요한지 몸으로 느끼게)
- "AI만 모르는 개발자"에 맞춘 눈높이 — 개발 기초는 설명 안 하고, AI 개념만 비유로 풀어줌

## 기술 스택 (코드 사전 스캔)
- Python 3.12, LangChain, Ollama (DeepSeek R1:8b), ChromaDB, nomic-embed-text
- MCP는 이후 챕터에서 추가 예정
